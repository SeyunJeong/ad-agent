# 광고 품질 스코어링 루브릭

> **적용 대상**: 모든 캠페인 크리에이티브 (Phase 5 QA Scoring에서 평가)
> **평가자**: QA_SCORER (주), AD_FACT_CHECKER (Data Accuracy), CREATIVE_REVIEWER (Legal Compliance)
> **만점**: 50점 (5개 지표 × 10점)

---

## 스코어링 체계

### 총점 기준

| 등급 | 총점 | 개별 최저 | 판정 | 액션 |
|------|------|----------|------|------|
| **PASS** | >= 35 | >= 5 | 통과 | Phase 6 (Channel Setup) 진행 |
| **REVISE** | >= 30 | 또는 개별 < 5 | 수정 | Phase 3 (Production) 리턴 |
| **REJECT** | < 30 | 또는 개별 < 3 | 거절 | Phase 2 (Concept) 리턴 |

> **필수 게이트**: Data Accuracy가 FAIL이면 총점 무관하게 REJECT

---

## 5대 평가 지표

### 1. Strategy Alignment (전략 정합성) — QA_SCORER

| 점수 | 기준 |
|------|------|
| **9-10** | 전략 USP가 모든 채널 크리에이티브에 일관되게 반영. 타겟 페르소나 핵심 니즈에 정확히 대응. Self-Debate 결론이 크리에이티브에 완벽 반영 |
| **7-8** | 전략 USP 대부분 반영. 일부 채널에서 메시지 일관성 부족하나 핵심 방향은 유지 |
| **5-6** | 전략 방향은 맞으나 USP 반영이 피상적. 타겟 니즈와 크리에이티브 간 갭 존재 |
| **3-4** | 전략과 크리에이티브 간 괴리. USP가 불명확하거나 왜곡. Self-Debate 결론 미반영 |
| **1-2** | 전략 무시. 크리에이티브가 전략 방향과 무관하거나 상충 |

**체크포인트**:
- [ ] Self-Debate 최종 전략의 USP가 헤드라인에 반영되었는가?
- [ ] Red Team이 지적한 약점이 크리에이티브에서 보완되었는가?
- [ ] 채널별 크리에이티브가 동일한 전략 메시지를 전달하는가?
- [ ] 타겟 페르소나의 언어/톤이 카피에 반영되었는가?

---

### 2. Data Accuracy (데이터 정확성) — AD_FACT_CHECKER

| 점수 | 기준 |
|------|------|
| **9-10** | 모든 수치/주장이 검증 가능한 출처와 함께 확인됨. 통계 인용 정확. 비교 표현 객관적 |
| **7-8** | 대부분 검증 완료. 1-2건 출처 불명확하나 사실관계 자체는 정확 |
| **5-6** | 주요 수치는 정확하나 일부 과장 표현 또는 맥락 없는 통계 인용 |
| **3-4** | 검증 불가 수치 다수. 과장/왜곡 표현 존재. 출처 미제시 |
| **1-2** | 허위 또는 오해 유발 수치. 검증 실패 항목 다수. **즉시 REJECT** |

**체크포인트**:
- [ ] 모든 수치에 출처가 명시되었는가?
- [ ] "1위", "최고", "유일" 등 최상급 표현에 근거가 있는가?
- [ ] 비교 광고 시 비교 기준이 공정하고 명확한가?
- [ ] 할인율/가격 표시가 정확한가?
- [ ] 통계 데이터의 기준 시점이 명시되었는가?

**FAIL 조건 (즉시 REJECT)**:
- 허위 수치 사용
- 검증 불가능한 효능/효과 주장
- 법적으로 금지된 표현 사용 (예: 의료기기의 치료 효과 주장)

---

### 3. Creative Impact (크리에이티브 임팩트) — QA_SCORER

| 점수 | 기준 |
|------|------|
| **9-10** | 스크롤 멈추게 하는 힘. 즉각적 주목 + 명확한 가치 전달. 감정적 공감 + 행동 유도 완벽 |
| **7-8** | 강한 주목도. 가치 제안 명확. CTA 효과적이나 약간의 개선 여지 |
| **5-6** | 평균적 주목도. 메시지는 전달되나 차별화 부족. CTA 존재하나 임팩트 약함 |
| **3-4** | 주목도 낮음. 경쟁 광고 대비 차별점 없음. CTA 불명확 또는 약함 |
| **1-2** | 무시당할 수준. 메시지 불명확. CTA 없거나 혼란 유발 |

**체크포인트**:
- [ ] 3초 내 핵심 메시지가 전달되는가?
- [ ] 경쟁 광고 대비 시각적/언어적 차별점이 있는가?
- [ ] 감정적 훅(호기심/공감/놀라움/긴급성)이 작동하는가?
- [ ] CTA가 명확하고 즉각적 행동을 유도하는가?
- [ ] A/B 테스트 가설이 측정 가능한가?

---

### 4. Channel Optimization (채널 최적화) — QA_SCORER

| 점수 | 기준 |
|------|------|
| **9-10** | 채널별 포맷/톤/길이 완벽 준수. 플랫폼 네이티브 느낌. 사용자 행동 패턴에 최적화 |
| **7-8** | 채널 규격 준수. 대부분 네이티브 느낌이나 1-2개 채널에서 미세 조정 필요 |
| **5-6** | 기본 규격은 맞으나 채널 특성 미반영. 동일 카피를 채널별로 단순 변환한 느낌 |
| **3-4** | 채널 규격 일부 미준수. 톤/길이 부적절. 플랫폼 정책 위반 가능성 |
| **1-2** | 채널 규격 무시. 광고 게재 불가 수준 |

**채널별 체크포인트**:

**네이버**:
- [ ] 파워링크 제목 15자/설명 45자 이내?
- [ ] 쇼핑 제목 최적화 (검색어 포함)?
- [ ] 검색 의도(탐색/비교/구매)에 맞는 톤?

**카카오**:
- [ ] 비즈보드 25자/디스플레이 45자 이내?
- [ ] 카카오톡 내 자연스러운 톤?
- [ ] 메시지 광고 시 개인화 변수 활용?

**메타 (인스타/페이스북)**:
- [ ] 피드 헤드라인 25자/본문 125자 권장?
- [ ] 릴스/스토리 세로형 최적화?
- [ ] 시각 중심 + 최소 텍스트?

**구글**:
- [ ] RSA 헤드라인 30자×3 / 설명 90자×2?
- [ ] 디스플레이 다양한 사이즈 대응?
- [ ] 유튜브 5초 스킵 전 핵심 전달?

---

### 5. Legal Compliance (법적 준수) — CREATIVE_REVIEWER

| 점수 | 기준 |
|------|------|
| **9-10** | 모든 법적/정책 요건 충족. 필수 고지사항 포함. 채널 광고정책 완전 준수 |
| **7-8** | 법적 요건 충족. 사소한 고지사항 위치/크기 조정 필요 |
| **5-6** | 주요 법적 요건 충족하나 채널 정책 일부 미준수. 수정으로 해결 가능 |
| **3-4** | 법적 위험 존재. 과장 광고, 고지사항 누락, 정책 위반 |
| **1-2** | 명백한 법률 위반. 허위/기만 광고. **즉시 REJECT** |

**체크포인트**:
- [ ] 표시광고법 준수? (허위/과장/기만/부당비교 없음)
- [ ] 업종별 필수 고지사항 포함? (금융: 위험고지, 건강: 개인차고지, 식품: 기능성 표시)
- [ ] 채널별 광고 정책 준수? (네이버/카카오/메타/구글 각각)
- [ ] 개인정보 수집 관련 고지 적절?
- [ ] 경품/할인 조건 명확히 표시?

**즉시 REJECT 조건**:
- 허위/기만 광고 (표시광고법 위반)
- 의료법 위반 표현
- 아동 대상 부적절 표현
- 타사 상표권 침해

## 스코어카드 출력 형식

```markdown
# QA Score Card

## 캠페인: {campaign_name}
## 평가일: {date}
## 이터레이션: #{iteration}

| # | 지표 | 평가자 | 점수 | 판정 |
|---|------|--------|------|------|
| 1 | Strategy Alignment | QA_SCORER | {n}/10 | {PASS/WARN/FAIL} |
| 2 | Data Accuracy | AD_FACT_CHECKER | {n}/10 | {PASS/WARN/FAIL} |
| 3 | Creative Impact | QA_SCORER | {n}/10 | {PASS/WARN/FAIL} |
| 4 | Channel Optimization | QA_SCORER | {n}/10 | {PASS/WARN/FAIL} |
| 5 | Legal Compliance | CREATIVE_REVIEWER | {n}/10 | {PASS/WARN/FAIL} |

**총점: {total}/50**
**최종 판정: {PASS / REVISE / REJECT}**

### 상세 피드백
{지표별 개선 포인트}

### 리턴 지시 (REVISE/REJECT 시)
- 리턴 대상: Phase {2 or 3}
- 수정 요청 사항: {구체적 수정 지시}
- 참조: {관련 전략/리서치 데이터}
```

---

## 리턴 루프 규칙

1. **최대 리턴 횟수**: 3회 (3회 초과 시 TEAM_LEAD 에스컬레이션)
2. **리턴 시 전달 데이터**: 스코어카드 + 구체적 수정 지시 + 원본 전략/리서치 참조
3. **REVISE → Phase 3**: Production 팀에 수정 지시, 기존 컨셉 유지
4. **REJECT → Phase 2**: Concept Planner에 재기획 요청, 전략은 유지
5. **매 리턴마다**: 이전 스코어카드를 참조하여 동일 문제 반복 방지
