# 리서치 사전 검증 체크리스트

> **적용 대상**: DEEP_RESEARCHER (Phase 0)
> **목적**: 리서치 시작 전 5개 게이트를 통과해야 본 리서치 진행
> **원칙**: 잘못된 리서치 방향은 전체 파이프라인을 오염시킨다

---

## 5개 사전 검증 게이트

### Gate 1: 브리프 완결성 (Brief Completeness)

**통과 조건**: 캠페인 브리프의 필수 필드가 모두 존재

| 필수 필드 | 설명 | 부재 시 액션 |
|----------|------|-------------|
| business_name | 사업체/브랜드명 | TEAM_LEAD에 요청 |
| product_service | 제품/서비스 설명 | TEAM_LEAD에 요청 |
| industry | 업종/카테고리 | 추론 가능하면 추론, 아니면 요청 |
| objective | 캠페인 목표 | TEAM_LEAD에 요청 |
| target_audience | 타겟 오디언스 | 기본 프로파일링 후 확인 요청 |
| budget | 예산 규모 | TEAM_LEAD에 요청 |
| channels | 희망 채널 | 전 채널 리서치 후 추천 |

**판정**:
- 필수 5개 (business_name, product_service, objective, target_audience, budget) 중 3개 이상 부재 → **BLOCK** (리서치 불가, TEAM_LEAD에 브리프 보완 요청)
- 1-2개 부재 → **WARN** (추론 가능한 항목은 추론, 리서치 진행하되 가정 명시)
- 전부 존재 → **PASS**

---

### Gate 2: 리서치 범위 정의 (Scope Definition)

**통과 조건**: 리서치할 범위와 깊이가 명확히 정의됨

**필수 정의 항목**:

```markdown
## 리서치 스코프
- 시장 규모: {TAM/SAM/SOM 추정 범위}
- 경쟁사 범위: {직접 경쟁 N개 + 간접 경쟁 N개}
- 트렌드 기간: {최근 N개월/년}
- 지역 범위: {국내/국내+해외}
- 채널 범위: {리서치 대상 채널}
```

**판정**:
- 범위 무한정 확대 (예: "전 세계 모든 경쟁사") → **BLOCK** (범위 축소 필요)
- 범위 정의 가능하나 일부 모호 → **WARN** (합리적 범위로 자체 정의, 가정 명시)
- 명확히 정의됨 → **PASS**

---

### Gate 3: 기존 데이터 확인 (Existing Data Check)

**통과 조건**: 이미 보유한 데이터/인사이트를 확인하여 중복 리서치 방지

**확인 대상**:
1. `archive/completed_campaigns.md` — 동일/유사 업종 과거 캠페인
2. `simulation/` — 과거 시뮬레이션 데이터
3. `config/templates.md` — 업종별 템플릿 (이미 벤치마크 포함)
4. `config/kpi_definitions.md` — KPI 벤치마크 (이미 채널별 데이터 포함)
5. 이전 이터레이션 리서치 (feedback loop에서 재진입 시)

**판정**:
- 기존 데이터 무시하고 처음부터 리서치 → **WARN** (기존 데이터 먼저 활용)
- 기존 데이터 확인 후 갭 분석 완료 → **PASS**

---

### Gate 4: 출처 신뢰도 기준 (Source Reliability Standard)

**통과 조건**: 리서치에 사용할 출처의 신뢰도 기준이 설정됨

**출처 신뢰도 등급**:

| 등급 | 출처 유형 | 활용 기준 |
|------|----------|----------|
| **A (공식)** | 정부 통계, 플랫폼 공식 자료, 상장사 IR | 무조건 신뢰, 인용 필수 |
| **B (전문)** | 업계 리서치 기관 (닐슨, 와이즈앱 등), 전문 미디어 | 신뢰, 발행일 확인 |
| **C (참고)** | 블로그, 커뮤니티, SNS, 비공식 리포트 | 참고만, 교차 검증 필수 |
| **D (비신뢰)** | 출처 불명, 광고성 콘텐츠, 오래된 데이터 (2년+) | 사용 금지 |

**판정**:
- A/B 등급 출처 계획 없이 C/D만 의존 → **BLOCK**
- A/B 70% 이상 + C 보조 → **PASS**

---

### Gate 5: 시간/리소스 적정성 (Time/Resource Adequacy)

**통과 조건**: 리서치 범위 대비 할당 시간이 적절

**기준**:

| 리서치 유형 | 예상 소요 | 최소 출처 수 |
|------------|----------|-------------|
| 간단 (단일 업종, 국내) | 10-15분 | 5개 |
| 표준 (복수 경쟁사, 국내) | 20-30분 | 10개 |
| 심층 (크로스 업종, 국내+해외) | 30-45분 | 15개 |
| 풀 리서치 (신규 시장 진입) | 45-60분 | 20개 |

**판정**:
- 범위 대비 시간 부족 → **WARN** (범위 축소 또는 우선순위 설정)
- 적정 → **PASS**

---

## 체크리스트 출력 형식

```markdown
# Research Pre-Check

## 캠페인: {campaign_name}
## 날짜: {date}

| # | 게이트 | 판정 | 비고 |
|---|--------|------|------|
| 1 | Brief Completeness | {PASS/WARN/BLOCK} | {부재 필드 또는 가정} |
| 2 | Scope Definition | {PASS/WARN/BLOCK} | {정의된 범위} |
| 3 | Existing Data Check | {PASS/WARN} | {활용 가능 데이터} |
| 4 | Source Reliability | {PASS/BLOCK} | {계획 출처 등급} |
| 5 | Time/Resource | {PASS/WARN} | {예상 소요} |

**최종 판정: {GO / CONDITIONAL GO / NO-GO}**

### 가정 목록 (WARN 항목)
{가정한 내용과 검증 계획}
```

**판정 기준**:
- BLOCK 1개 이상 → **NO-GO** (TEAM_LEAD에 보고, 해결 후 재시작)
- WARN 3개 이상 → **CONDITIONAL GO** (가정 명시하고 진행, 결과물에 가정 리스트 첨부)
- WARN 2개 이하 → **GO** (리서치 진행)
