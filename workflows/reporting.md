# 리포팅 워크플로우 (Type C)

## 개요
캠페인 성과 데이터를 수집하고 분석하여 인사이트를 도출하고, 오너가 의사결정에 활용할 수 있는 리포트를 생성하는 워크플로우.
ANALYTICS_TEAM(DATA_MONITOR + INSIGHT_ANALYST) 주도로 데이터 수집 → 정합성 확인 → 분석 → 인사이트 도출 → 리포트 작성 → 오너 보고.

---

## 트리거

| 조건 | 예시 | 판단 기준 |
|------|------|----------|
| 정기 리포트 | 매주 월요일, 매월 초 | 정기 스케줄 |
| 수시 리포트 요청 | "이번 주 성과 알려줘" | 오너 직접 요청 |
| 캠페인 종료 | 캠페인 기간 종료 | 최종 성과 보고 |
| 전략 회의 자료 | "내일 회의 자료 준비해줘" | 의사결정 지원 |
| 벤치마크 비교 | "우리 CTR 괜찮은 편이야?" | 업종 비교 분석 |

## 프로세스

```
[오너 요청 또는 정기 스케줄]
         ↓
[TEAM_LEAD] 접수 + 리포트 유형 판단
  → 주간/월간/분기/수시/캠페인 종료
  → 리포트 범위 결정 (전체/채널별/캠페인별)
         ↓
[Phase 1: 데이터 수집] (DATA_MONITOR + CHANNEL_TEAM)
  → DATA_MONITOR: 통합 데이터 수집 주도
  → 각 채널 전문가: 채널별 성과 데이터 API 수집 지원
    ├── NAVER_SPECIALIST: 네이버 광고 관리 시스템 데이터
    ├── KAKAO_SPECIALIST: 카카오 모먼트 대시보드 데이터
    ├── META_SPECIALIST: Meta Ads Manager 데이터
    └── GOOGLE_SPECIALIST: Google Ads + GA4 데이터
  → DATA_MONITOR: 데이터 정합성 확인 (채널별 수치 크로스 체크)
  → 산출물: 정제된 원본 데이터 세트
         ↓
[Phase 2: 데이터 분석] (INSIGHT_ANALYST)
  → 핵심 KPI 계산
  → 전기(전주/전월) 대비 변화 분석
  → 목표 대비 달성률 계산
  → 채널별 비교 분석
  → 캠페인/키워드/소재별 Top/Bottom 분석
  → 어트리뷰션 분석 (멀티 채널)
  → 산출물: 분석 결과 데이터
         ↓
[Phase 3: 인사이트 도출] (INSIGHT_ANALYST + CAMPAIGN_STRATEGIST)
  → INSIGHT_ANALYST: 성과 변화 원인 분석 (왜 올랐는지/떨어졌는지)
  → INSIGHT_ANALYST: 기회/위협 식별
  → INSIGHT_ANALYST: 업종 벤치마크 대비 평가
  → CAMPAIGN_STRATEGIST: 전략적 인사이트 보강 (필요 시)
  → 다음 기간 권고사항 도출
  → 산출물: 인사이트 + 권고사항
         ↓
[Phase 4: 리포트 작성] (INSIGHT_ANALYST)
  → 리포트 유형에 맞는 양식으로 작성
  → 경영진 요약 (1페이지)
  → 상세 분석 (채널별/캠페인별)
  → 시각화 데이터 (표/차트 형태)
  → 권고사항 + 다음 액션
  → 산출물: 완성 리포트
         ↓
[Phase 5: 검수] (TEAM_LEAD)
  → 데이터 정확성 확인 (합계, 계산)
  → 인사이트 논리성 확인
  → 권고사항 실현 가능성 확인
  → 오탈자/포맷 확인
         ↓
[Phase 6: finance 연동] (DATA_MONITOR)
  → 광고비 집행 현황 데이터 추출
  → finance 팀 전달 (handoff/budget_to_finance.md)
         ↓
[TEAM_LEAD] 오너에게 리포트 제출
  → 리포트 + 핵심 인사이트 요약 + 권고사항
```

## 리포트 유형별 상세

### 1. 주간 리포트 (매주 월요일)

```
### 1. 한 줄 요약
{이번 주 핵심 성과를 한 문장으로}

### 2. 핵심 지표 (vs 전주)
| 지표 | 이번 주 | 전주 | 변화 | 목표 | 달성률 |
|------|---------|------|------|------|--------|
| 광고비 | | | | | |
| 노출수 | | | | | |
| 클릭수 | | | | | |
| CTR | | | | | |
| CPC | | | | | |
| 전환수 | | | | | |
| CPA | | | | | |
| 매출 | | | | | |
| ROAS | | | | | |

### 3. 채널별 성과
| 채널 | 담당 전문가 | 광고비 | 전환 | CPA | ROAS |
|------|-----------|--------|------|-----|------|
| 네이버 | NAVER_SPECIALIST | | | | |
| 카카오 | KAKAO_SPECIALIST | | | | |
| 메타 | META_SPECIALIST | | | | |
| 구글 | GOOGLE_SPECIALIST | | | | |

### 4. Top 3 / Bottom 3
- Top: {고성과 키워드/소재/캠페인}
- Bottom: {저성과 키워드/소재/캠페인}

### 5. 주요 변화 및 원인
- {변화 1}: {원인}

### 6. 다음 주 액션
```

**작성 담당**: INSIGHT_ANALYST (DATA_MONITOR 데이터 기반 + CAMPAIGN_STRATEGIST 전략 인사이트)

```

## 기간: {년}년 {월}월

### 1. 경영진 요약

### 2. 월간 핵심 지표 (vs 전월)
```

## 데이터 정합성 체크리스트 (DATA_MONITOR 담당)

```
[ ] 채널별 데이터 기간 일치 (동일 기간 비교)
[ ] 노출 > 클릭 > 전환 (역전 없음)
[ ] 비용 합계 = 채널별 비용 합산
[ ] 전환수: 채널 리포트 vs GA4 전환 비교
[ ] ROAS 계산: 매출 / 광고비 (세금/수수료 처리 기준 명시)
[ ] 전주/전월 데이터: 동일 기준 동일 기간
[ ] 소수점 처리: CTR, CVR 등 소수점 둘째자리까지
[ ] 통화 단위: 원(원) 통일
```

## 에러 처리

### 데이터 불일치
```
{채널A}와 {채널B/GA4}의 전환 수가 {N}건 차이납니다.
원인: {가능한 원인 - 어트리뷰션 기준 차이, 트래킹 지연, 픽셀 오류 등}
대응: {데이터 기준 명시하고 양쪽 수치 모두 기재}
확인 담당: {해당 채널 전문가} + DATA_MONITOR
```

### 데이터 지연
```
{채널}의 {기간} 데이터가 아직 반영되지 않았습니다.
예상 반영 시간: {시간}
대응: 반영 후 리포트 업데이트 예정
확인 담당: {해당 채널 전문가}
```
