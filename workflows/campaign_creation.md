# 캠페인 생성 워크플로우 (Type A)

## 개요
새로운 캠페인을 처음부터 설계하여 실행 가능한 캠페인 플랜을 만드는 워크플로우.
비즈니스 목표 분석 → 전략 수립 (셀프 디베이트) → 키워드/오디언스 리서치 (서브팀 병렬) → 크리에이티브 (파이프라인 + 법적 검수) → 입찰/예산 → 채널 세팅 (독립 실행) → 모니터링 설정까지 전 과정을 커버한다.

---

## 트리거

| 조건 | 예시 | 판단 기준 |
|------|------|----------|
| 신규 캠페인 요청 | "캠페인 만들어줘", "광고 시작하고 싶어" | 처음 시작하는 새 캠페인 |
| 신규 채널 진출 | "네이버 광고 해보고 싶어" | 기존에 없던 채널 추가 |
| 신규 제품/서비스 론칭 | "신제품 광고 플랜 짜줘" | 새 제품의 마케팅 |
| 프로모션/이벤트 캠페인 | "블프 세일 캠페인 짜줘" | 시즌/이벤트 캠페인 |

## 프로세스

```
[오너: 캠페인 요청]
         ↓
[TEAM_LEAD] 접수 + 브리프 작성/확인
  → 캠페인 목표/예산/타겟/채널 파악
  → 누락 정보 오너에게 질문
         ↓
[Phase 1: 전략 수립] (CAMPAIGN_STRATEGIST)
  → 비즈니스 목표 분석
  → 채널 믹스 설계
  → 예산 배분 전략
  → KPI 목표 설정
  → **셀프 디베이트 (Blue Team → Red Team → 보완)**
  → 산출물: 캠페인 전략서 (디베이트 로그 포함)
  → TEAM_LEAD 검수
         ↓
[Phase 2: 키워드/오디언스 리서치] (KEYWORD_AUDIENCE_TEAM)
  → 병렬 리서치 실행:
    ├── SEARCH_RESEARCHER: 네이버/구글 키워드 분석
    ├── AUDIENCE_RESEARCHER: 카카오/메타 오디언스 세그먼트 설계
    └── TREND_ANALYST: 시즌 트렌드 + 경쟁사 인텔리전스
  → 크로스 리뷰 (상호 검토)
  → SEARCH_RESEARCHER 리드로 통합 보고서 작성
  → 산출물: 통합 키워드/오디언스 리서치 보고서
  → TEAM_LEAD 검수
         ↓
[Phase 3: 크리에이티브 제작] (CREATIVE_TEAM)
  → 병렬 제작:
    ├── COPYWRITER: 채널별 광고 카피 세트 + A/B 변형
    └── VISUAL_DIRECTOR: 크리에이티브 가이드 + thread-team 소재 요청
  → 크로스 리뷰 (메시지 정합성 확인)
  → CREATIVE_REVIEWER: 법적/규제/정책 검수 (PASS/REVISE/REJECT)
  → 산출물: 검수 완료된 광고 카피 세트 + 크리에이티브 가이드
  → TEAM_LEAD 검수
         ↓
[Phase 4: 입찰/예산 설계] (BID_OPTIMIZER)
  → 채널별 입찰 전략 설정
  → 예산 페이싱 계획 수립
  → 비용 예측
  → 산출물: 입찰 전략서 + 예산 페이싱 계획
  → TEAM_LEAD 검수
  → finance 연동 (예산 승인 필요 시)
         ↓
[Phase 5: 채널 세팅] (CHANNEL_TEAM - 각 채널 독립 실행)
  → 각 채널 전문가가 독립적으로 세팅:
    ├── NAVER_SPECIALIST: 네이버 검색/쇼핑/브랜드 세팅
    ├── KAKAO_SPECIALIST: 카카오 모먼트/메시지 세팅
    ├── META_SPECIALIST: 메타/인스타 세팅 + 픽셀/CAPI 설정
    └── GOOGLE_SPECIALIST: 구글 검색/디스플레이/P-Max 세팅
  → 크로스 채널 동기화 (메시지 일관성, UTM 규칙)
  → 론칭 체크리스트 확인
  → 산출물: 채널별 세팅 완료 보고
  → TEAM_LEAD 검수
         ↓
[Phase 6: 최종 검수 + 모니터링 설정] (TEAM_LEAD + ANALYTICS_TEAM)
  → 전체 캠페인 플랜 정합성 확인
  → DATA_MONITOR: 트래킹 정상 작동 확인 + 모니터링 대시보드 설정
  → INSIGHT_ANALYST: KPI 모니터링 기준 설정
  → 산출물: 최종 캠페인 플랜
         ↓
[TEAM_LEAD] 오너에게 최종 보고
  → 캠페인 전략 요약 (셀프 디베이트 결과 포함)
  → 채널별 실행 계획
  → 예산 배분표
  → KPI 목표
  → 리스크/주의사항 (Red Team 검증 결과)
  → 론칭 일정
```

## 단계별 상세

### Step 0: 브리프 확인 (TEAM_LEAD)
**입력**: 오너의 요청
**처리**:
- 캠페인 브리프 필수 항목 확인
- 누락 항목 오너에게 질문
- 브리프 구조화

**필수 확인 항목**:
```
[ ] 비즈니스/제품/서비스 설명
[ ] 캠페인 목표 (인지도/트래픽/전환)
[ ] 타겟 오디언스
[ ] 총 예산 및 기간
[ ] 선호 채널 (있을 경우)
[ ] 경쟁사 정보 (있을 경우)
[ ] 랜딩 페이지 URL
[ ] 기존 소재 존재 여부
[ ] 특별 요청사항
```

### Step 1: 전략 수립 (CAMPAIGN_STRATEGIST)
**입력**: 구조화된 캠페인 브리프
**처리**: 비즈니스 분석 → 채널 믹스 → 예산 배분 → KPI → **셀프 디베이트**
**출력**: 캠페인 전략서 (디베이트 로그 포함)
**핸드오프**: `handoff/strategy_to_keyword_team.md`

**전략서 필수 포함 항목**:
```
[ ] 캠페인 목표 및 KPI
[ ] 타겟 오디언스 프로파일
[ ] 채널 믹스 및 채널별 역할
[ ] 예산 배분 (채널별/기간별)
[ ] 핵심 메시지 및 USP
[ ] 퍼널 전략
[ ] 경쟁 분석 요약
[ ] 리스크 평가 (Red Team 검증 결과)
[ ] 셀프 디베이트 로그
```

### Step 2: 키워드/오디언스 리서치 (KEYWORD_AUDIENCE_TEAM)
**입력**: 캠페인 전략서 (`handoff/strategy_to_keyword_team.md`)
**처리**:
```
Phase 1 - 병렬 리서치 (동시 실행):
├── SEARCH_RESEARCHER → 네이버/구글 키워드 리스트
├── AUDIENCE_RESEARCHER → 카카오/메타 오디언스 세그먼트
└── TREND_ANALYST → 시즌 트렌드 + 경쟁사 인텔리전스

Phase 2 - 크로스 리뷰 (상호 검토):
└── 각 리서처가 다른 리서처의 결과물을 검토/피드백

Phase 3 - 통합 (SEARCH_RESEARCHER 리드):
└── 최종 통합 리서치 보고서 작성
```
**출력**: 통합 키워드/오디언스 리서치 보고서
**핸드오프**: `handoff/keyword_team_to_creative_team.md`

### Step 3: 크리에이티브 제작 (CREATIVE_TEAM)
**입력**: 전략서 + 통합 리서치 보고서 (`handoff/keyword_team_to_creative_team.md`)
**처리**:
```
Phase 1 - 병렬 제작:
├── COPYWRITER → 채널별 카피 세트 + A/B 변형
└── VISUAL_DIRECTOR → 크리에이티브 가이드 + thread-team 소재 요청

Phase 2 - 크로스 리뷰:
└── COPYWRITER ↔ VISUAL_DIRECTOR: 메시지/비주얼 정합성

Phase 3 - 법적/정책 검수:
└── CREATIVE_REVIEWER → PASS/REVISE/REJECT (최대 2회 피드백 루프)

Phase 4 - 최종 패키징:
└── 검수 완료 소재 패키지
```
**출력**: 검수 완료된 광고 카피 세트 + 크리에이티브 가이드
**핸드오프**: `handoff/creative_team_to_channel_team.md`

**thread-team 연동 판단**:
```
이미지/영상 소재 필요 → VISUAL_DIRECTOR가 thread-team에 요청
  → handoff/creative_to_thread.md 양식으로 전달
  → thread-team 제작 → VISUAL_DIRECTOR 1차 검수 → CREATIVE_REVIEWER 2차 검수
카피만 필요 → COPYWRITER 자체 처리
```

**CREATIVE_REVIEWER 품질 게이트** (필수):
```
[ ] 표시광고법 준수 (과대광고, 허위광고)
[ ] 식약처 규정 (건강기능식품, 의약품)
[ ] 금감원 규정 (금융 상품)
[ ] 채널별 광고 정책 (네이버/카카오/메타/구글)
[ ] 브랜드 가이드라인 준수
판정: PASS → 다음 단계 / REVISE → 수정 후 재심사 / REJECT → 전면 재제작
```

### Step 4: 입찰/예산 설계 (BID_OPTIMIZER)
**입력**: 전략서 + 키워드 CPC 데이터 (SEARCH_RESEARCHER 제공)
**처리**: 입찰 전략 → 예산 페이싱 → 비용 예측
**출력**: 입찰 전략서 + 예산 페이싱 계획

**finance 연동 판단**:
```
월 예산 500만원 이상 → finance 팀 승인 필요
  → handoff/budget_to_finance.md 양식으로 전달
월 예산 500만원 미만 → TEAM_LEAD 승인으로 진행
```

### Step 5: 채널 세팅 (CHANNEL_TEAM - 독립 실행)
**입력**: 전략서 + 키워드/오디언스 + 크리에이티브 + 입찰 전략 (`handoff/creative_team_to_channel_team.md`)
**처리**:
```
독립 실행 (각 채널 전문가):
├── NAVER_SPECIALIST: 파워링크/쇼핑검색/브랜드검색 세팅
├── KAKAO_SPECIALIST: 카카오 모먼트/메시지 세팅
├── META_SPECIALIST: 메타 캠페인 + 픽셀/CAPI 세팅
└── GOOGLE_SPECIALIST: 구글 캠페인 + GA4 연동

크로스 채널 동기화:
├── UTM 파라미터 규칙 통일
├── 트래킹 코드 충돌 확인
├── 론칭 시간 동기화
└── 채널 간 메시지 일관성 확인
```
**출력**: 채널별 세팅 완료 보고 + 론칭 체크리스트

### Step 6: 최종 검수 + 모니터링 설정 (TEAM_LEAD + ANALYTICS_TEAM)
**입력**: 전체 산출물
**처리**:
```
TEAM_LEAD: 전체 정합성 확인
DATA_MONITOR:
├── 트래킹 정상 작동 확인 (모든 채널)
├── 대시보드 설정
├── 이상 감지 기준 설정
└── 일일 스냅샷 수집 시작
INSIGHT_ANALYST:
├── KPI 모니터링 기준 설정
├── 첫 주간 리포트 일정 확정
└── 벤치마크 기준 설정
```
**출력**: 최종 캠페인 플랜 → 오너 보고

## 에러 처리

### 브리프 정보 부족
```
캠페인 설계를 위해 추가 정보가 필요합니다:
1. {누락 항목 1}
2. {누락 항목 2}
이 정보가 있으면 더 정확한 전략을 세울 수 있어요.
```

### 예산-목표 불일치
```
요청하신 목표({목표})를 달성하기 위한 최소 예산은 {금액}원입니다.
현재 예산({금액}원)으로 가능한 옵션:
1. 목표 하향 조정: {조정된 목표}
2. 채널 축소: {N}개 채널로 집중
3. 기간 단축: {N}주로 테스트 먼저
```

### 크리에이티브 검수 실패
```
CREATIVE_REVIEWER 검수 결과: {REVISE/REJECT}
사유: {구체적 사유}
수정 대상: {수정 필요 항목}
재심사 예정: {시간}
```

### 채널 세팅 이슈
```
{채널 전문가}가 다음 이슈를 보고했습니다:
- 이슈: {상세}
- 영향: {영향 범위}
- 대응: {해결 방안}
- 예상 소요: {시간}
```

## 산출물 템플릿

### 최종 캠페인 플랜 (오너 보고용)
```
# 캠페인 플랜: {캠페인명}

## 1. 캠페인 개요
- 목표: {목표}
- 기간: {시작일} ~ {종료일}
- 총 예산: {금액}원

## 2. 전략 요약 (셀프 디베이트 결과 반영)
- 핵심 메시지: {메시지}
- 타겟: {타겟 요약}
- 채널 구성: {채널 목록}
- Red Team 검증 결과: {주요 리스크 + 보완 내용}

## 3. 채널별 실행 계획
| 채널 | 담당 전문가 | 광고 유형 | 예산 | KPI |
|------|-----------|----------|------|-----|
{채널별 상세 계획}

## 4. 예산 배분
{채널별/기간별 예산 표}

## 5. KPI 목표
{채널별 KPI 목표 표}

## 6. 크리에이티브 미리보기
{채널별 주요 카피/소재 미리보기}
- 법적 검수 상태: {CREATIVE_REVIEWER PASS 확인}

## 7. 일정
{론칭까지 타임라인}

## 8. 리스크 및 대응
{주요 리스크 + 대응 방안 (Red Team 검증)}

## 9. 모니터링 계획
- DATA_MONITOR: 일일 스냅샷 + 이상 감지
- INSIGHT_ANALYST: 주간 리포트 (매주 월요일)
- 첫 성과 리포트: 론칭 후 {N}일

## 10. 다음 단계
- 오너 승인 후 론칭
- 론칭 후 {N}일 뒤 첫 성과 리포트
- 주간 최적화 사이클 시작
```
