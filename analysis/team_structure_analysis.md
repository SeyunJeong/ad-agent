## 1. 분석 개요 및 방법론

### 1.1 현재 구조

현재 AdAgent는 TEAM_LEAD 1명 + 6개 역할 에이전트(각 1명)로 구성된 7인 체제다.
각 에이전트가 해당 도메인 전체를 단독으로 처리한다.

```
[TEAM_LEAD]
    |
    +-- CAMPAIGN_STRATEGIST (1)
    +-- KEYWORD_RESEARCHER (1)
    +-- AD_CREATIVE (1)
    +-- BID_OPTIMIZER (1)
    +-- ANALYTICS_AGENT (1)
    +-- CHANNEL_SPECIALIST (1)
```

### 1.2 검토 질문

각 역할에 대해 다음을 분석한다:

1. 해당 역할의 업무 범위가 단일 에이전트로 충분히 커버되는가?
2. 팀으로 전환 시 어떤 구체적 이점이 있는가?
3. 멀티 에이전트의 조정 오버헤드가 이점을 상쇄하지 않는가?
4. 한국 시장 특수성이 팀 전환을 요구하는가?

### 1.3 분석 방법

- 한국 주요 퍼포먼스 에이전시(에코마케팅, 메조미디어, 나스미디어, 제일기획, 퍼포먼스바이TBWA, 이노션) 조직 구조 리서치
- 멀티 에이전트 AI 시스템 최신 연구 검토
- 현재 AdAgent 에이전트 정의서 분석

## 2. 한국 퍼포먼스 에이전시 조직 벤치마크

### 2.1 에코마케팅 (국내 No.1 퍼포먼스 마케팅 그룹, 매출 4,430억원)

에코마케팅은 **UNIT 조직 + 챕터 조직** 이중 구조를 채택하고 있다.
이는 AI 에이전트 팀 설계에 가장 직접적인 참고가 된다.

| 조직 축 | 팀 구성 | 역할 |
|---------|--------|------|
| **산업별 UNIT** | Commerce, Beauty, Finance, Ent., Global Biz, New Biz | 산업 도메인 전문성 |
| **크리에이티브 챕터** | Video Production, Image Creatives, Landing Page, Copywrite | 소재 유형별 전문 제작 |
| **매체별 챕터** | Facebook, Instagram, Google, Kakao, Naver, New Media | 채널별 운영 전문성 |
| **데이터/기술 챕터** | Data Science, Marketing Tech, UX Consulting, GA Consulting | 분석 및 기술 |

**핵심 인사이트**: 에코마케팅은 하나의 캠페인에 산업 UNIT + 매체 챕터 + 크리에이티브 챕터가 **크로스 펑셔널**로 협업한다. 단일 담당자가 모든 것을 하지 않는다. 특히 크리에이티브와 매체 운영은 명확히 세분화되어 있다.

### 2.2 제일기획 BOUNCE(바운스)팀

제일기획은 퍼포먼스 마케팅 전문 부서 '바운스(BOUNCE)팀'을 별도 신설했다.

- **DnA센터**: 데이터 마케팅 전략 조직. 디지털 데이터 실시간 수집/분석, 자동화 마케팅 플랫폼 운영
- **바운스팀**: 광고 소재 최적화 + 검색 광고 전략 수립/운영 + NFT 퍼포먼스 + 라이브 커머스 + 캠페인 대시보드

**핵심 인사이트**: 전략/데이터(DnA센터)와 실행/최적화(바운스팀)가 분리되어 있다. 하나의 팀 안에서도 소재 최적화, 검색 광고, 대시보드가 별도 기능으로 존재한다.

### 2.3 퍼포먼스바이TBWA (89명, AI-Driven AdTech)

- **AI 기반 3대 플랫폼**: Addirect(TV 광고), Aitive(AI 영상 제작), Abi(미디어 성과 최적화)
- **서비스 구조**: AI 소재 제작 ~ 캠페인 운영 ~ 최적화까지 Full Coverage

**핵심 인사이트**: 크리에이티브 제작과 미디어 성과 최적화를 별도 플랫폼으로 분리하여 각각을 AI로 자동화하고 있다. 이는 해당 영역이 별도 전문 시스템을 필요로 할 만큼 복잡하다는 반증이다.

### 2.4 이노션

- **AI솔루션팀**: 2024년 3월 신설, 생성형 AI 기반 솔루션 전담
- **이노션에스(INNOCEAN S)**: 소셜 마케팅 전문 법인, 국내/글로벌 솔루션 2개 본부
- **디퍼플, 디플랜360 인수**: 디지털 퍼포먼스, 미디어렙 영역 별도 법인

**핵심 인사이트**: 이노션은 소셜 마케팅, 퍼포먼스 마케팅, 미디어 바잉을 각각 별도 법인으로 분리할 정도로 전문성이 다르다고 판단하고 있다.

### 2.5 메조미디어 (CJ ENM 디지털 마케팅 그룹)

- 2026 트렌드 리포트에서 강조: AI 마케팅과 발견형 커머스가 핵심 트렌드
- 광고 집행 프로세스에서 자동화 가능 범위가 급속 확대 중
- "상품과 목표만 있으면 AI가 소재, 타겟, 입찰, 성과 측정을 자동 처리"

**핵심 인사이트**: 2026년 기준 AI 자동화가 소재 생성, 타겟팅, 입찰 최적화를 포괄적으로 커버할 수준에 도달했다. 다만 전략 수립과 최종 판단은 여전히 인간(또는 상위 에이전트) 영역이다.

### 2.6 한국 광고 대행사 표준 직무 구분

한국 광고 대행사의 일반적 직무 분류:

| 직무 | 역할 | 비고 |
|------|------|------|
| **AE (Account Executive)** | 광고주 소통, 전체 프로젝트 총괄, 팀 간 조율 | 소규모 대행사에서는 AE가 전략/미디어플랜까지 겸임 |
| **AP (Account Planner)** | 캠페인 전략 수립, 시장 분석 | AE에서 전략 기능 분리 |
| **미디어 플래너 (MP)** | 매체 전략, 예산 배분, 미디어 믹스 | 매체사 대응, 미디어 바잉 |
| **옵티마이저** | 매체별 성과 최적화, 입찰 관리 | 매체별로 전담 배치가 일반적 |
| **크리에이티브 디렉터 (CD)** | 크리에이티브 총괄, 톤앤매너 결정 | 카피라이터, 디자이너 관리 |
| **카피라이터** | 광고 문구 작성 | 채널별 규격 숙지 필요 |
| **디자이너** | 이미지/배너/영상 소재 제작 | 이미지/영상 분리가 일반적 |
| **데이터 분석가** | GA, 대시보드, 리포팅 | 최근 수요 급증 |

**종합 인사이트**: 대형 에이전시일수록 역할이 세분화되어 있으며, 특히 (1) 크리에이티브 영역(카피/이미지/영상 분리)과 (2) 채널 운영 영역(매체별 전담자 배치)에서 팀화가 두드러진다.

## 3. 멀티 에이전트 시스템 원칙

### 3.1 팀화가 가치 있는 경우

최신 연구(2025-2026)에 따르면, 멀티 에이전트 시스템이 단일 에이전트보다 우수한 경우:

1. **다양한 관점이 필요한 경우**: 서로 다른 페르소나가 동일 문제를 바라보면 맹점을 줄일 수 있다
2. **병렬 처리가 가능한 경우**: 독립적으로 수행 가능한 하위 작업이 있을 때
3. **전문 도메인이 명확히 구분되는 경우**: 네이버 전문가와 메타 전문가는 필요한 지식 세트가 다르다
4. **검증/비평 루프가 필요한 경우**: 제작자와 검수자를 분리하면 품질이 향상된다

### 3.2 팀화를 피해야 하는 경우

1. **조정 오버헤드가 이점을 초과하는 경우**: 에이전트 3-4개를 넘으면 조정 비용이 초선형(지수 1.724)으로 증가한다
2. **도구 의존 작업**: 도구 집약적 작업에서 멀티 에이전트는 단일 에이전트 대비 2-6배 비효율적
3. **단순 파이프라인 작업**: 입력→처리→출력의 단순 흐름은 단일 에이전트가 효율적
4. **컨텍스트가 긴밀하게 연결된 작업**: 분산 시 컨텍스트 손실이 크면 단일이 유리

### 3.3 설계 원칙

- **팀 내 에이전트 수 제한**: 최적 3-4명 (연구 근거)
- **명확한 역할 분리**: 역할 중복 최소화
- **협업 모델 명시**: 병렬 리서치→토론, 순차 파이프라인, 제작→검수 중 적합한 모델 선택
- **점진적 확장**: MVP(최소) → 필요 시 확장

## 4. 역할별 상세 분석

---

### 4.1 Campaign Strategist (캠페인 전략가)

#### 4.1.1 현재 단일 에이전트 커버 범위

- 비즈니스 목표 분석 → 마케팅 전략 변환
- 채널 믹스 설계 (4개 채널 동시 고려)
- 예산 배분 전략
- 퍼널 설계 (인지→고려→전환)
- 경쟁사 분석
- 한국 시장 시즌/트렌드 반영

#### 4.1.2 팀이 필요한가?

**결론: 팀 전환 권장하지 않음. 단일 에이전트 유지 + 보조 기능 강화.**

**이유**:
- 전략 수립은 **통합적 사고**가 핵심. 채널 믹스, 퍼널 설계, 예산 배분을 하나의 관점에서 통합해야 한다
- 한국 대형 에이전시에서도 전략 수립은 AP(Account Planner) 1명 또는 소수 시니어가 총괄한다
- 전략가를 여러 명으로 나누면 전략의 일관성이 깨질 위험이 크다
- 멀티 에이전트 연구에서 "전략적 의사결정은 단일 에이전트가 더 효과적"이라는 결론이 지배적

**단, 보조적 토론 메커니즘은 유의미하다**:
- 전략 초안 작성 후, Critic(비평) 에이전트가 약점을 지적하는 방식
- 이는 별도 팀이 아니라, 전략가 에이전트의 **셀프 디베이트 프롬프트**로 구현 가능

#### 4.1.3 권고 구조

```
[CAMPAIGN_STRATEGIST] (단일 에이전트, 기존 유지)
    |
    +-- 내부 셀프 디베이트 모드 (전략 초안 → 자기 비평 → 수정)
    +-- TEAM_LEAD의 전략 검수 기능 강화
```

| 구분 | 에이전트 수 |
|------|-----------|
| **MVP** | 1명 (현행 유지) |
| **이상적** | 1명 + 셀프 디베이트 모드 활성화 |

#### 4.1.4 협업 모델

- 기존과 동일: TEAM_LEAD로부터 브리프 수령 → 전략 수립 → 하위 에이전트에 배분
- 변경점: 전략 수립 시 "레드팀(Red Team) 관점" 프롬프트를 추가하여 전략의 약점을 스스로 점검

### 4.2 Keyword/Audience Researcher (키워드/오디언스 리서처)

#### 4.2.1 현재 단일 에이전트 커버 범위

- 네이버 키워드 리서치 (검색량, 경쟁도, CPC)
- 구글 키워드 리서치 (Keyword Planner)
- 카카오 모먼트 오디언스 세그먼트 설계
- 메타 오디언스 세그먼트 설계
- 경쟁 키워드 분석
- 검색 의도 분류
- 시즌/트렌드 분석
- 네거티브 키워드 전략

#### 4.2.2 팀이 필요한가?

**결론: 팀 전환 강력 권장. 3명 구성.**

**이유**:

1. **플랫폼별 전문성 차이가 극심하다**:
   - 네이버 키워드 리서치와 메타 오디언스 설계는 **완전히 다른 도메인**이다
   - 네이버는 키워드 기반(검색 의도, 자동완성, 연관검색어), 메타는 관심사/행동 기반(오디언스 세그먼트, 룩어라이크)
   - 한국 시장에서 네이버 검색 트래픽이 55% 이상을 차지하므로 네이버 전문성이 특히 중요

2. **병렬 리서치→토론 모델이 자연스럽다**:
   - 각 리서처가 독립적으로 리서치한 후, 결과를 종합/토론하면 더 나은 키워드 세트가 도출된다
   - 에코마케팅도 매체별 팀(Naver Team, Google Team 등)이 별도로 리서치한 결과를 통합한다

3. **리서치 범위가 방대하다**:
   - 시드 키워드 도출 → 확장 → 분석 → 스코어링 → 네거티브 도출을 각 채널별로 수행하면 작업량이 4배
   - 현재 에이전트 정의서에서도 "채널당 최소 50개, 최대 500개 키워드" 요구

#### 4.2.3 제안 팀 구성

```
[KEYWORD_AUDIENCE_TEAM]
    |
    +-- 검색 리서처 (Search Researcher)
    |   : 네이버 + 구글 검색 키워드 전문
    |   : 검색 의도 분류, 시드 키워드 확장, CPC/경쟁도 분석
    |   : 네이버 데이터랩, 블랙키위 활용
    |
    +-- 소셜/오디언스 리서처 (Social/Audience Researcher)
    |   : 카카오 + 메타 오디언스 세그먼트 전문
    |   : 관심사/행동 타겟팅, 룩어라이크 설계
    |   : 커스텀 오디언스 전략
    |
    +-- 트렌드/경쟁 분석가 (Trend & Competitive Analyst)
        : 시즌 트렌드, 경쟁사 키워드/광고 분석
        : 블루오션 키워드 발굴
        : 웹 서치 기반 실시간 트렌드 포착
```

#### 4.2.4 협업 모델: 병렬 리서치 → 크로스 리뷰 → 통합

```
Phase 1: 병렬 리서치 (동시 실행)
    검색 리서처 → 네이버/구글 키워드 리스트 초안
    소셜 리서처 → 카카오/메타 오디언스 세그먼트 초안
    트렌드 분석가 → 시즌 트렌드 + 경쟁사 인텔리전스

Phase 2: 크로스 리뷰 (순차 실행)
    각 리서처가 다른 리서처의 결과물을 검토하고 피드백
    예: 검색 리서처가 발견한 고볼륨 키워드를 소셜 리서처가
        오디언스 관점에서 검증 ("이 키워드의 검색자가 메타에서
        도달 가능한가?")

Phase 3: 통합 보고서 작성
    검색 리서처가 통합 키워드/오디언스 리포트를 작성
    (리드 역할 겸임)
```

#### 4.2.5 규모 권고

| 구분 | 에이전트 수 | 구성 |
|------|-----------|------|
| **MVP** | 2명 | 검색 리서처 + 소셜/오디언스 리서처 (트렌드 분석은 두 리서처가 분담) |
| **이상적** | 3명 | 검색 + 소셜/오디언스 + 트렌드/경쟁 분석가 |

### 4.3 Ad Creative Generator (광고 크리에이티브)

#### 4.3.1 현재 단일 에이전트 커버 범위

- 채널별 광고 카피 작성 (네이버/카카오/메타/구글, 각각 2-4개 하위 포맷)
- 한국어 네이티브 카피라이팅
- 크리에이티브 가이드 제작 (이미지/영상 방향)
- A/B 테스트 변형 생성
- USP 도출
- thread-team 연동 (소재 요청)

#### 4.3.2 팀이 필요한가?

**결론: 팀 전환 강력 권장. 가장 팀화 효과가 큰 역할. 3-4명 구성.**

**이유**:

1. **크리에이티브 유형별 전문성이 명확히 다르다**:
   - 에코마케팅은 크리에이티브를 Video Production / Image Creatives / Landing Page / Copywrite 4개 팀으로 분리
   - 퍼포먼스바이TBWA도 AI 영상 제작(Aitive)을 별도 플랫폼으로 운영
   - 카피라이팅과 비주얼 디렉션은 다른 스킬셋

2. **채널별 포맷이 완전히 다르다**:
   - 네이버 파워링크: 15자 제목 + 45자 설명 (검색 의도 매칭 중심)
   - 메타 리일스: 1초 후킹 + 시각적 스토리텔링 (감성/엔터테인먼트)
   - 구글 RSA: 15개 제목 변형 + 4개 설명 (조합 최적화)
   - 이 모든 것을 한 에이전트가 동시에 높은 퀄리티로 하기 어렵다

3. **제작→검수 분리가 품질을 높인다**:
   - 카피 작성자와 법적/규제 검수자를 분리하면 과대광고 위험이 줄어든다
   - 한국은 표시광고법, 식약처 규정, 금융감독원 규정 등이 까다롭다
   - 멀티 에이전트 연구에서 "Research Agent + Creative Agent + Critic Agent" 구조가 크리에이티브 그룹씽크를 방지한다고 입증됨

4. **A/B 테스트 변형의 다양성**:
   - 여러 에이전트가 독립적으로 변형을 생성하면, 단일 에이전트의 패턴 고착 문제를 해결할 수 있다

#### 4.3.3 제안 팀 구성

```
[CREATIVE_TEAM]
    |
    +-- 카피라이터 (Copywriter)
    |   : 모든 채널의 텍스트 카피 전담
    |   : 한국어 네이티브 카피, USP 메시지화
    |   : 네이버(15자/45자), 카카오(25자/34자), 메타(125자),
    |     구글 RSA(30자x15개) 등 채널별 제약 완벽 숙지
    |   : A/B 테스트용 카피 변형 생성
    |
    +-- 비주얼 디렉터 (Visual Director)
    |   : 이미지/영상 크리에이티브 가이드 제작
    |   : 채널별 규격 관리 (사이즈, 비율, 텍스트 비율)
    |   : thread-team 연동 (크리에이티브 브리프 작성)
    |   : 소재 트렌드 리서치 (Meta Ad Library, 경쟁사 분석)
    |
    +-- 크리에이티브 심사관 (Creative Reviewer / Critic)
    |   : 법적/규제 검수 (표시광고법, 식약처, 금감원)
    |   : 브랜드 일관성 검수
    |   : 채널 정책 적합성 확인
    |   : A/B 가설의 논리적 타당성 검증
    |   : 카피 자연스러움/오탈자 최종 점검
    |
    +-- (이상적 추가) 퍼포먼스 크리에이티브 분석가 (Performance Creative Analyst)
        : 기존 소재 성과 데이터 분석
        : 고성과 소재의 패턴 추출 ("이 USP + 이 포맷이 CTR 3% 이상")
        : 데이터 기반 신규 소재 방향 제안
        : ANALYTICS_AGENT와 긴밀 협업
```

#### 4.3.4 협업 모델: 순차 파이프라인 + 피드백 루프

```
Phase 1: 브리프 수령
    CAMPAIGN_STRATEGIST → 핵심 메시지/USP
    KEYWORD_RESEARCHER → 주요 키워드 및 검색 의도

Phase 2: 병렬 제작
    카피라이터 → 채널별 카피 세트 초안
    비주얼 디렉터 → 크리에이티브 가이드 초안

Phase 3: 크로스 리뷰
    카피라이터 <-> 비주얼 디렉터: 카피와 비주얼의 메시지 정합성 확인
    심사관 → 카피/비주얼 모두 법적/정책 검수
    (피드백 → Phase 2 반복, 최대 2회)

Phase 4: 최종 패키징
    카피라이터가 최종 크리에이티브 세트 패키징
    CHANNEL_SPECIALIST에게 전달
```

#### 4.3.5 규모 권고

| 구분 | 에이전트 수 | 구성 |
|------|-----------|------|
| **MVP** | 2명 | 카피라이터 + 심사관 (비주얼은 카피라이터가 가이드 수준으로 겸임) |
| **이상적** | 3명 | 카피라이터 + 비주얼 디렉터 + 심사관 |
| **풀 스케일** | 4명 | 위 3명 + 퍼포먼스 크리에이티브 분석가 |

### 4.4 Bid/Budget Optimizer (입찰/예산 최적화)

#### 4.4.1 현재 단일 에이전트 커버 범위

- 채널별 입찰 전략 설계 (수동/자동/목표 기반)
- 예산 페이싱 (일/주/월)
- ROAS/CPA 최적화
- 채널 간 예산 재배분
- 시즌/이벤트 입찰 전략
- 경쟁 입찰 분석

#### 4.4.2 팀이 필요한가?

**결론: 팀 전환 권장하지 않음. 단일 에이전트 유지.**

**이유**:

1. **입찰 최적화는 고도로 수학적/알고리즘적이다**:
   - 입찰 전략 설계는 데이터 기반의 규칙 적용이 핵심
   - 여러 에이전트가 토론할 주관적 영역이 적다
   - "목표 CPA가 15,000원이고 현재 18,000원이면 입찰가를 X% 낮춘다"는 논리적 연산

2. **통합 관점이 필수적이다**:
   - 채널 간 예산 재배분은 4개 채널을 **동시에** 보면서 결정해야 한다
   - 네이버에서 빼서 메타에 넣는 판단은 두 채널의 상대적 효율을 비교해야 하므로, 한 에이전트가 전체를 보는 것이 유리

3. **도구 집약적 작업이다**:
   - 멀티 에이전트 연구에서 도구 의존 작업은 단일 에이전트가 2-6배 효율적이라는 결과
   - 입찰 최적화는 각 채널 API를 통한 데이터 조회/설정이 핵심

4. **에이전시 벤치마크**:
   - 대부분의 에이전시에서도 미디어 플래너(전략)와 옵티마이저(실행)를 분리하지만, 옵티마이저 내부를 더 세분화하지는 않는다
   - 채널별 옵티마이저를 두는 경우가 있지만, 이는 채널 전문가(Channel Specialist)의 역할에 가깝다

#### 4.4.3 권고 구조

```
[BID_OPTIMIZER] (단일 에이전트, 기존 유지)
    |
    +-- 자동 알림 규칙 강화 (예산 과소진/미소진 자동 감지)
    +-- ANALYTICS_AGENT와의 데이터 파이프라인 강화
```

| 구분 | 에이전트 수 |
|------|-----------|
| **MVP** | 1명 (현행 유지) |
| **이상적** | 1명 (기능 강화) |

#### 4.4.4 참고

입찰 최적화가 팀이 아닌 이유 중 하나는, 메조미디어가 지적한 대로 이 영역이 **AI 자동화가 가장 빠르게 진행되는 분야**이기 때문이다. 네이버/구글/메타 모두 자체 자동 입찰 알고리즘(스마트 비딩)을 제공하며, 에이전트의 역할은 "어떤 자동 입찰 전략을 선택하고, 목표를 어떻게 설정할 것인가"라는 메타 판단에 집중된다. 이 메타 판단은 1명의 전문 에이전트로 충분하다.

### 4.5 Analytics Agent (성과 분석)

#### 4.5.1 현재 단일 에이전트 커버 범위

- 성과 대시보드 설계 (경영진 요약, 채널별, 캠페인별)
- KPI 추적 (CTR, CPC, CPA, ROAS, CVR, CPM 등)
- 채널별 비교 분석
- 어트리뷰션 분석 (멀티 터치)
- 주간/월간 리포트 생성
- 이상 감지 및 알림
- finance 팀 연동 (광고비 리포트)

#### 4.5.2 팀이 필요한가?

**결론: 제한적 팀 전환 권장. 2명 구성.**

**이유**:

1. **실시간 모니터링과 심층 분석은 다른 작업이다**:
   - 실시간 성과 모니터링 + 이상 감지 = 반복적, 규칙 기반, 빈번한 실행
   - 심층 인사이트 분석 + 전략 제안 = 비정기적, 창의적 사고, 깊은 분석
   - 에코마케팅도 Data Science, GA Consulting을 분리

2. **리포팅과 분석은 다르다**:
   - 리포트 작성 = 데이터 수집 → 정형화된 포맷으로 변환 (자동화 가능)
   - 인사이트 도출 = "왜 이런 결과인가" + "어떻게 해야 하는가" (전문적 판단)
   - 한국 에이전시에서도 리포팅 담당과 시니어 분석가가 분리되는 추세

3. **다만, 3명 이상은 과도하다**:
   - 분석 영역은 컨텍스트 공유가 매우 중요 (데이터 해석의 일관성)
   - 분석가가 많으면 같은 데이터에 대해 상충하는 해석이 나올 위험

#### 4.5.3 제안 팀 구성

```
[ANALYTICS_TEAM]
    |
    +-- 데이터 모니터 (Data Monitor)
    |   : 실시간 성과 모니터링, 이상 감지/알림
    |   : 일일 성과 스냅샷 자동 생성
    |   : 데이터 수집/정리/대시보드 업데이트
    |   : finance 팀 연동 (광고비 데이터 자동 전달)
    |   : KPI 건강 상태 판정 (정상/주의/위험)
    |
    +-- 인사이트 분석가 (Insight Analyst)
        : 주간/월간 리포트 작성 (인사이트 중심)
        : 크로스 채널 어트리뷰션 분석
        : 성과 원인 분석 ("왜 CPA가 올랐는가?")
        : 데이터 기반 전략 제안 (CAMPAIGN_STRATEGIST에게)
        : A/B 테스트 결과 통계적 분석
        : 업종 벤치마크 대비 분석
```

#### 4.5.4 협업 모델: 계층적 분업

```
상시 루프 (데이터 모니터):
    데이터 수집 → 대시보드 업데이트 → 이상 감지 → 알림
    (반복 주기: 일 1-2회 또는 실시간)

정기 분석 (인사이트 분석가):
    데이터 모니터가 정리한 데이터 기반 → 심층 분석 → 리포트 작성
    (주기: 주 1회 주간 리포트, 월 1회 월간 리포트)

비정기 분석 (인사이트 분석가):
    이상 감지 시 → 원인 분석 → 대응 제안
    A/B 테스트 완료 시 → 통계적 유의성 분석 → 결론 도출
```

#### 4.5.5 규모 권고

| 구분 | 에이전트 수 | 구성 |
|------|-----------|------|
| **MVP** | 1명 | 기존 유지 (모니터링과 분석 겸임) |
| **이상적** | 2명 | 데이터 모니터 + 인사이트 분석가 |

### 4.6 Channel Specialist (채널 전문가)

#### 4.6.1 현재 단일 에이전트 커버 범위

- 네이버 검색광고 전문 (파워링크, 쇼핑검색, 브랜드검색, 파워콘텐츠)
- 카카오 모먼트 전문 (디스플레이, 메시지, 비즈보드)
- 메타 광고 전문 (피드, 스토리/리일스, DPA)
- 구글 광고 전문 (검색, 디스플레이, YouTube, 쇼핑, P-Max)
- 채널별 API 연동
- 채널 정책 준수
- 트러블슈팅

#### 4.6.2 팀이 필요한가?

**결론: 팀 전환 가장 강력 권장. 가장 명확한 팀화 근거가 있는 역할. 최소 2명, 이상적 4명.**

**이유**:

1. **채널별 전문성이 완전히 다르다 - 한국 시장의 핵심 이슈**:
   - 네이버 검색광고: 한국 고유 플랫폼, 검색 알고리즘/정책/심사 기준이 독자적
   - 카카오 모먼트: 카카오톡 생태계(채널, 친구톡, 알림톡) 고유 지식 필요
   - 메타: 글로벌 플랫폼이지만 iOS14+ 이슈, CAPI 설정 등 기술적 전문성 필요
   - 구글: Search, Display, YouTube, Shopping, P-Max 각각이 소규모 도메인
   - **에코마케팅이 Facebook Team, Instagram Team, Google Team, Kakao Team, Naver Team을 각각 별도 팀으로 운영하는 것이 이를 증명한다**

2. **API 연동 스펙이 채널마다 완전히 다르다**:
   - 네이버 검색광고 API, 카카오 모먼트 API, Meta Marketing API, Google Ads API
   - 각 API의 인증 방식, 엔드포인트 구조, 제약사항이 모두 다르다
   - 하나의 에이전트가 4개 API를 모두 깊이 있게 다루기 어렵다

3. **정책/심사 기준이 채널마다 다르다**:
   - 네이버: 상표 키워드 이의제기, 파워콘텐츠 콘텐츠 품질 가이드
   - 카카오: 채널 친구 기반 메시지 정책, 차단율 관리
   - 메타: iOS14+ ATT, 특수 카테고리 설정, 텍스트 비율 가이드
   - 구글: 상표 정책(한국), 건강/의약 인증, 아동 보호

4. **한국 시장의 현실**:
   - "같은 퍼포먼스 마케팅이라도 네이버 검색광고와 메타 광고는 완전히 다른 영역"
   - 국내 대행사들이 매체별 전문 조직을 별도로 보유하는 것이 업계 표준

#### 4.6.4 협업 모델: 독립 실행 + 크로스 채널 동기화

```
일상 운영 (독립적):
    각 채널 전문가가 자기 채널의 캠페인 세팅/실행/최적화를 독립적으로 수행
    채널별 체크리스트에 따라 론칭 전 자체 검수

크로스 채널 동기화 (정기적):
    주 1회 채널 간 성과 비교 회의 (동기화 포인트)
    - 채널 간 메시지 일관성 확인
    - 크로스 채널 어트리뷰션 데이터 공유
    - 채널 간 예산 재배분 필요성 검토
    → BID_OPTIMIZER 및 ANALYTICS_AGENT와 연동

트러블슈팅 (비정기):
    특정 채널 이슈 발생 시 해당 전문가가 즉시 대응
    크로스 채널 영향이 있으면 관련 전문가와 협의
```

#### 4.6.5 규모 권고

| 구분 | 에이전트 수 | 구성 |
|------|-----------|------|
| **MVP** | 2명 | 검색 전문가(네이버+구글) + 소셜 전문가(카카오+메타) |
| **이상적** | 4명 | 네이버 + 카카오 + 메타 + 구글 각각 1명 |

**MVP에서 2명으로 묶는 논리**:
- 네이버/구글은 모두 "검색 광고"가 핵심이므로 공통 지식이 많다
- 카카오/메타는 모두 "소셜/디스플레이"가 핵심이므로 유사점이 있다
- 단, 이 2명 체제에서도 각자 2개 채널의 고유 정책을 깊이 있게 관리해야 하므로 부담이 크다

## 6. 구현 우선순위 및 로드맵

### 6.1 Phase 1: 즉시 팀화 (최우선)

**Channel Specialist → Channel Team (2명)**

- **이유**: 채널별 전문성 차이가 가장 명확하고, 팀화 효과가 가장 직접적
- **리스크**: 낮음. 각 전문가의 역할이 독립적이므로 조정 오버헤드 최소
- **작업**: 기존 CHANNEL_SPECIALIST.md를 NAVER_SPECIALIST.md + GOOGLE_SPECIALIST.md / KAKAO_SPECIALIST.md + META_SPECIALIST.md로 분리 (또는 SEARCH_CHANNEL.md + SOCIAL_CHANNEL.md)

### 6.2 Phase 2: 크리에이티브 팀화

**Ad Creative → Creative Team (2명)**

- **이유**: 카피 작성과 법적/정책 검수 분리는 품질 향상에 직접적 효과
- **리스크**: 중간. 카피라이터-심사관 간 피드백 루프 설계 필요
- **작업**: AD_CREATIVE.md를 COPYWRITER.md + CREATIVE_REVIEWER.md로 분리

### 6.3 Phase 3: 리서치 팀화

**Keyword Researcher → Research Team (2명)**

- **이유**: 검색 키워드와 소셜 오디언스는 다른 도메인. 병렬 리서치 효과
- **리스크**: 중간. 리서치 결과 통합 프로세스 설계 필요
- **작업**: KEYWORD_RESEARCHER.md를 SEARCH_RESEARCHER.md + AUDIENCE_RESEARCHER.md로 분리

### 6.4 Phase 4: 분석 팀 분리 (선택적)

**Analytics Agent → Analytics Team (2명)**

- **이유**: 실시간 모니터링과 심층 분석 분리
- **리스크**: 낮음, 단 에이전트 수 대비 효과가 Phase 1-3보다 작을 수 있음
- **작업**: ANALYTICS_AGENT.md를 DATA_MONITOR.md + INSIGHT_ANALYST.md로 분리

### 6.5 비용-효과 분석

| Phase | 추가 에이전트 | 조정 오버헤드 | 기대 효과 | ROI 판정 |
|-------|-------------|-------------|----------|---------|
| Phase 1 | +1명 | 낮음 | 채널별 품질 50%+ 향상 | **매우 높음** |
| Phase 2 | +1명 | 중간 | 크리에이티브 품질/법적 안전성 향상 | **높음** |
| Phase 3 | +1명 | 중간 | 키워드 커버리지/정확도 향상 | **높음** |
| Phase 4 | +1명 | 낮음 | 분석 깊이/속도 향상 | **중간** |

### 6.6 주의사항

1. **팀 내 에이전트 수는 3-4명을 넘지 않도록 한다**: 연구에 따르면 에이전트 수가 4개를 넘으면 조정 오버헤드가 초선형으로 증가한다

2. **팀 내부 통신 프로토콜을 명확히 정의한다**: 각 팀에 대해 handoff 문서, 입출력 포맷, 리뷰 절차를 사전에 설계한다

3. **점진적으로 전환한다**: Phase 1부터 시작하여 효과를 측정한 후 다음 단계로 이동한다

4. **단일 에이전트가 유리한 역할은 무리하게 팀화하지 않는다**: CAMPAIGN_STRATEGIST와 BID_OPTIMIZER는 단일 에이전트가 더 효과적이라는 판단을 존중한다
