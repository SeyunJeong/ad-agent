
## A-2. INSIGHT_ANALYST: 분석 체계 설계

### 1. KPI 모니터링 기준

#### 1-1. 핵심 KPI 정의

| KPI | 정의 | 목표 | 측정 소스 | 보고 주기 |
|-----|------|------|-----------|-----------|
| 설치 수 (Installs) | Google Play 앱 설치 완료 건수 | 1,000건 / 2주 | GA4 first_open + 채널 보고 | 일일 |
| CPI (Cost Per Install) | 총 광고비 / 총 설치 수 | 3,000원 이하 | 채널별 집계 | 일일 |
| 활성화율 (Activation Rate) | 첫 링크 저장 / 설치 수 x 100 | 30% 이상 | GA4 first_link_save | 일일 |
| CTR (Click-Through Rate) | 클릭 수 / 노출 수 x 100 | 채널별 상이 | 채널별 보고 | 일일 |
| 채널별 CPI 비교 | 각 채널의 설치당 비용 | 최저 효율 채널 식별 | 채널별 집계 | 주간 |

#### 1-2. 보조 KPI

| KPI | 정의 | 목표 | 용도 |
|-----|------|------|------|
| DAU (Daily Active Users) | 일일 활성 사용자 수 | 설치의 40%+ | 리텐션 초기 지표 |
| D1 리텐션 | 설치 다음날 재방문율 | 30%+ | 앱 품질 판단 |
| 세션당 링크 저장 수 | 평균 링크 저장 횟수 | 2건+ | 제품 PMF 지표 |
| 노출당 비용 (CPM) | 광고비 / 노출 x 1000 | 참고용 | 채널 도달 효율 비교 |
| 보조 전환 | 라스트 클릭 외 기여 전환 | 참고용 | 크로스 채널 가치 평가 |

#### 1-3. 건강 상태 판정 기준

| KPI | 정상 | 주의 | 위험 |
|-----|------|------|------|
| CPI | < 3,000원 | 3,000~5,000원 | > 5,000원 |
| CTR (검색) | > 2.0% | 1.0~2.0% | < 1.0% |
| CTR (디스플레이) | > 0.8% | 0.5~0.8% | < 0.5% |
| CTR (소셜) | > 1.0% | 0.5~1.0% | < 0.5% |
| 활성화율 | > 30% | 15~30% | < 15% |
| 일일 설치 수 | > 71건 | 50~71건 | < 50건 |
| 예산 소진율 | 80~120% | 60~80% 또는 120~150% | < 60% 또는 > 150% |

### 2. 1주차 중간 분석 포인트 (D+7)

```
1주차 중간 분석 프레임워크:

■ 분석 목표: "어떤 채널이 가장 효율적인가?"

■ 분석 항목:

1. 채널별 CPI 비교
   ├── 메타 CPI vs 구글 CPI vs 네이버 CPI vs 카카오 CPI
   ├── 각 채널 CPI의 일별 추이 (학습 효과 확인)
   └── 채널별 설치 수 누적 현황

2. 채널 내 세부 분석
   ├── 네이버: 캠페인별 성과 (파워링크 vs 파워콘텐츠 vs GFA)
   │   └── 키워드별 성과 Top/Bottom 5
   ├── 카카오: 광고그룹별 성과 (IT관심사 vs 행동타겟 vs 비즈보드)
   ├── 메타: 광고세트별 성과 (IT직장인 vs 콘텐츠소비자 vs 브로드)
   │   └── 수동 캠페인 vs Advantage+ 성과 비교
   └── 구글: UAC 에셋별 등급 확인 (최고/양호/보통/미흡)
       └── 네트워크별 배분 현황 (검색/GDN/YouTube/Play)

3. 소재별 성과 분석
   ├── 메타: 이미지 vs 리일스 vs 캐러셀 CTR/CPI 비교
   ├── 네이버: 카피 A vs 카피 B 성과 비교
   └── 전 채널: USP별 메시지 효과 비교
       ├── "AI 자동 정리" 메시지 성과
       ├── "나중에 읽으려던 링크" 메시지 성과
       └── "북마크 정리" 메시지 성과

4. 타겟 오디언스 분석
   ├── 연령대별: 25-34 vs 35-44 CPI 비교
   ├── 성별: 성과 차이 유의성
   └── 관심사별: IT/테크 vs 일반 생산성 vs 콘텐츠 소비

5. 활성화율 분석
   ├── 채널별 활성화율(설치 → 첫 링크 저장) 비교
   ├── 활성화율이 높은 유입 경로 식별
   └── 낮은 활성화율 원인 가설 수립

6. 예산 소진 현황
   ├── 계획 대비 실제 소진율
   ├── 채널별 일예산 소진 패턴
   └── 잔여 예산으로 2주차 예산 재배분 제안

■ 액션 아이템 도출:
├── 고효율 채널: 예산 증액 제안 (금액 + 근거)
├── 저효율 채널: 원인 + 개선방안 또는 예산 감액 제안
├── 소재 교체: 저성과 소재 교체 요청 (AD_CREATIVE)
├── 타겟 조정: 비효율 타겟 세그먼트 축소/확대
└── 네이버 키워드: 저성과 키워드 제외, 고성과 키워드 입찰가 상향
```

```
   ├── 2차 목표: 활성화(첫 링크 저장) 달성률
   └── KPI 달성 vs 미달 상세 분해

2. 채널 효율 최종 비교
   ├── 채널별 CPI 최종 순위 + 신뢰구간
   ├── 채널별 CPM (도달 효율) 순위
   ├── 채널별 활성화율 순위 (양질의 유저 채널)
   ├── 채널별 ROI 종합 점수
   │   └── ROI 점수 = (설치 수 x 활성화율) / 광고비 x 100
   └── 채널별 역할 재평가
       ├── 인지 기여: 보조 전환 분석
       ├── 전환 기여: 라스트 클릭 전환
       └── 품질 기여: 활성화율

3. 크로스 채널 어트리뷰션 분석
   ├── GA4 기반 전환 경로 Top 10
   ├── 각 채널의 퍼스트 터치 / 라스트 터치 비교
   ├── 보조 전환 비율: 인지 채널(메타, YouTube)의 간접 기여
   └── 크로스 채널 시너지 효과 정량화
       예: "메타에서 인지 후 네이버 검색으로 전환" 경로 비중

4. 소재 최종 분석
   ├── 소재별 종합 성과 랭킹 (CTR x CVR)
   ├── 피로도 분석: 시간 경과에 따른 CTR 변화
   ├── 채널별 최적 소재 유형 도출
   └── 본 캠페인 소재 방향 제안

5. 타겟 오디언스 최종 분석
   ├── 코호트별 설치-활성화 전환 퍼널
   ├── 고가치 유저 프로필 도출 (설치 + 활성화 + D7 리텐션)
   ├── 채널별 유저 품질 비교
   └── 본 캠페인 타겟 전략 제안

6. 본 캠페인 예산 추정
   ├── 테스트 결과 기반 채널별 최적 예산 배분 비율
```

### 4. 채널 효율 비교 분석 방법론

```
채널 효율 비교 프레임워크:

■ 1단계: 단일 지표 비교 (기초)
├── CPI 비교: 가장 직관적인 효율 지표
├── CTR 비교: 소재/타겟팅 효율
├── CVR 비교: 전환 퍼널 효율
└── 주의: 단일 지표만으로 판단하면 안 됨

■ 2단계: 복합 지표 비교 (심화)
├── 비용 대비 양질 설치 = (설치 수 x 활성화율) / 광고비
├── 유저 품질 지수 = 활성화율 x D1 리텐션
├── 효율 지수 = (1 / CPI) x 활성화율 x 100
└── 채널별 유저 LTV 추정 (데이터 축적 후)

■ 3단계: 역할 기반 평가 (전략)
├── 인지 역할 채널: CPM + 도달수 + 보조 전환 기여로 평가
│   (메타, 카카오 비즈보드)
├── 전환 역할 채널: CPI + CVR + 라스트 클릭 기여로 평가
│   (네이버 검색, 구글 UAC)
└── 복합 역할 채널: 복합 지표로 평가
    (구글 UAC -- 검색 + GDN + YouTube 통합)

■ 4단계: 통계적 유의성 확인
├── 채널 간 CPI 차이의 95% 신뢰구간 산출
├── 샘플 크기 충분성 확인 (채널당 최소 50건 설치)
├── 외부 변수(시간대, 요일) 통제 여부
└── 결론: "통계적으로 유의한 차이" vs "데이터 부족으로 판단 보류"
```

### 5. A/B 테스트 유의성 판단 기준

```
A/B 테스트 분석 기준:

■ 테스트 대상:
├── 메타: 수동 캠페인 vs Advantage+ 캠페인 (CPI 비교)
├── 메타: 이미지 vs 리일스 vs 캐러셀 (소재 유형 테스트)
├── 네이버: 카피 A "AI 자동 정리" vs 카피 B "북마크 정리 끝" (CTR 비교)
└── 전 채널: USP 메시지 효과 비교

■ 유의성 판단 기준:
├── 유의수준: p < 0.05 (95% 신뢰수준)
├── 최소 샘플: 변형당 100건 클릭 (CTR 테스트) 또는 50건 전환 (CVR 테스트)
├── 테스트 기간: 최소 3일 (요일 효과 분산)
├── 트래픽 배분: 50:50 균등 배분 확인
└── 외부 변수: 시간대, 오디언스 동일 조건 확인

■ 결과 해석 기준:
├── p < 0.05 + 효과 크기 > 10%: "채택 권고" -- 실질적 개선
├── p < 0.05 + 효과 크기 < 10%: "조건부 채택" -- 통계적 유의하나 효과 미미
├── p >= 0.05 + 샘플 충분: "동등" -- 차이 없음, 다른 기준으로 선택
├── p >= 0.05 + 샘플 부족: "연장" -- 추가 데이터 필요
└── 방법: 이항 비율 z-검정 (CTR/CVR) 또는 Mann-Whitney U 검정 (CPI)

■ 판단 후 액션:
├── 승자 확정: 전체 예산을 승자 변형에 배분
├── 동등: 비용 효율이 높은 변형 선택 또는 다음 테스트 설계
└── 연장: 테스트 기간 3일 추가 연장
```
