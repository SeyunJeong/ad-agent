# KEYWORD/AUDIENCE RESEARCH TEAM - 팀 워크플로우

> **팀**: Keyword/Audience Research Team
> **구성원**: SEARCH_RESEARCHER (리드), AUDIENCE_RESEARCHER, TREND_ANALYST
> **버전**: 1.0
> **최종 수정**: 2026-02-17

---

## 1. 팀 개요

### 1.1 팀 미션

캠페인 전략에 기반하여 검색 키워드, 오디언스 세그먼트, 트렌드/경쟁 인텔리전스를
통합적으로 리서치하고, 크리에이티브 팀과 채널 팀이 즉시 활용할 수 있는
고품질 통합 리서치 보고서를 산출한다.

### 1.2 팀 구성 및 역할

| 역할 | 에이전트 | 주요 책임 | 리드 여부 |
|------|---------|----------|---------|
| 검색 리서처 | SEARCH_RESEARCHER | 네이버/구글 키워드 리서치, 통합 보고서 작성 | **리드** |
| 소셜/오디언스 리서처 | AUDIENCE_RESEARCHER | 카카오/메타 오디언스 세그먼트 설계 | - |
| 트렌드/경쟁 분석가 | TREND_ANALYST | 시즌 트렌드, 경쟁사 분석, 블루오션 발굴 | - |

### 1.3 리드 역할 (SEARCH_RESEARCHER)

- 팀 작업 시작 시 브리프를 팀원에게 배분
- 크로스 리뷰 일정 조율
- 세 리서처의 결과물을 통합하여 최종 보고서 편집
- 외부 팀(Creative Team, Channel Team)에 핸드오프 문서 작성
- 품질 최종 검수

## 3. Phase별 상세 가이드

### 3.1 Phase 1: 브리프 접수 및 배분

**담당**: SEARCH_RESEARCHER (리드)

```
입력 수령:
├── CAMPAIGN_STRATEGIST로부터 전략서 수령
│   (handoff/strategy_to_keyword_team.md 양식)
├── 브리프 확인 사항:
│   ├── 캠페인 목표 (전환/인지/트래픽)
│   ├── 타겟 오디언스 정의
│   ├── 대상 채널 목록
│   ├── 예산 범위
│   ├── 경쟁사 리스트
│   ├── 캠페인 기간
│   └── 제품/서비스 USP
└── 배분:
    ├── AUDIENCE_RESEARCHER에게: 페르소나 정보, 타겟 채널 (카카오/메타), 퍼스트파티 데이터 현황
    └── TREND_ANALYST에게: 경쟁사 리스트, 업종, 캠페인 기간 (시즌 확인용)
```

**브리프 불완전 시 처리**:
- 필수 정보 누락 시 CAMPAIGN_STRATEGIST에게 추가 요청
- 경쟁사 리스트가 없으면 TREND_ANALYST에게 경쟁사 식별부터 의뢰

### 3.2 Phase 2: 병렬 리서치

**세 에이전트가 동시에 독립적으로 리서치 수행**

| 에이전트 | 수행 작업 | 산출물 |
|---------|----------|--------|
| SEARCH_RESEARCHER | 네이버/구글 키워드 발굴, 분석, 스코어링 | 채널별 키워드 리스트 초안 |
| AUDIENCE_RESEARCHER | 카카오/메타 오디언스 세그먼트 설계 | 채널별 오디언스 설계서 초안 |
| TREND_ANALYST | 시즌 트렌드 + 경쟁사 분석 + 블루오션 발굴 | 트렌드/경쟁 분석 보고서 초안 |

**병렬 리서치 규칙**:
1. 각 에이전트는 자기 영역에 집중하되, 다른 에이전트에게 도움이 될 발견사항은 메모해둔다
2. 다른 에이전트의 작업이 완료되길 기다리지 않고 독립적으로 진행한다
3. 불확실한 사항은 "크로스 리뷰 시 확인" 태그를 달아둔다

### 3.3 Phase 3: 크로스 리뷰

**모든 에이전트가 다른 에이전트의 결과물을 자기 전문 관점에서 리뷰**

#### 크로스 리뷰 매트릭스

| 리뷰어 → 대상 | SEARCH 결과물 | AUDIENCE 결과물 | TREND 결과물 |
|-------------|-------------|----------------|-------------|
| **SEARCH** | (자기 것) | 오디언스 규모가 키워드 볼륨과 일관성 있는지, 검색 의도와 퍼널 매핑 정합성 | 트렌드 키워드가 실제 검색량으로 뒷받침되는지, 블루오션의 현실성 |
| **AUDIENCE** | 고볼륨 키워드의 검색자가 카카오/메타에서 도달 가능한지, 키워드에서 파생할 관심사 있는지 | (자기 것) | 경쟁사 타겟팅 추론 결과 반영 여부, 트렌드 기반 오디언스 확장 기회 |
| **TREND** | 경쟁사 핵심 키워드와 겹치는 Tier 1 확인, 차별화 포인트 제안 | 경쟁사 미진출 오디언스 기회 제안, 시즌별 오디언스 변화 반영 여부 | (자기 것) |

#### 크로스 리뷰 피드백 양식

```
[크로스 리뷰 피드백]
리뷰어: {에이전트명}
대상: {리뷰 대상 에이전트명}
날짜: {YYYY-MM-DD}

1. 긍정 피드백:
   - {잘 된 점}

2. 수정 제안:
   - {수정이 필요한 항목}: {수정 방향}
   - 근거: {데이터 또는 논리적 근거}

3. 추가 제안:
   - {놓친 키워드/오디언스/트렌드}
   - 근거: {발견 경로}

   - {확인이 필요한 질문}
```

### 3.4 Phase 4: 피드백 반영 및 수정

- 크로스 리뷰 피드백을 받은 에이전트는 타당한 피드백을 수용하여 결과물을 수정
- 의견이 충돌할 경우 SEARCH_RESEARCHER (리드)가 최종 판단
- **최대 1회 수정 사이클**로 제한 (무한 루프 방지)
- 수정 사항이 경미할 경우 Phase 5에서 통합 시 반영 가능

### 3.5 Phase 5: 통합 보고서 작성

**담당**: SEARCH_RESEARCHER (리드)

```
통합 보고서 구조:
├── 1. 리서치 개요

│   ├── 경쟁사 인텔리전스

│   └── 업종 인사이트
│
├── 5. 크로스 인사이트 (통합 관점)
│   ├── 키워드-오디언스 매핑
│   ├── 트렌드 기반 키워드/오디언스 조정
│   └── 경쟁 환경 기반 차별화 전략
│
└── 6. 권고사항 및 다음 단계
    ├── Creative Team 핸드오프 요약

    └── 주의사항 (법적/정책 이슈)
```

## 5. 품질 기준

### 5.1 팀 전체 품질 체크리스트

| # | 항목 | 기준 | 확인 주체 |

| 4 | 경쟁사 분석이 키워드/오디언스에 반영되었는가 | 차별화 전략이 키워드 선정에 영향 | 리드 |

| 8 | 통합 보고서에 누락이 없는가 | 3명의 결과물이 모두 포함 | 리드 |

| 10 | 법적/정책 리스크가 표시되었는가 | 상표권, 개인정보 이슈 | 각자 |

### 5.2 품질 게이트

```
Phase 2 → Phase 3 게이트:
├── 각 에이전트의 초안이 최소 기준을 충족하는지 리드가 빠르게 확인
├── 키워드 수 최소 기준 충족 여부
├── 오디언스 세그먼트 수 최소 기준 충족 여부
└── 경쟁사 분석 수 최소 기준 충족 여부

Phase 4 → Phase 5 게이트:
├── 크로스 리뷰 피드백이 모두 반영되었는지 확인
├── 해결되지 않은 이슈가 없는지 확인
└── 팀 내 의견 충돌이 해소되었는지 확인
```

## 6. 예외 처리

### 6.1 브리프 불완전

```
필수 정보 누락 시:
├── 타겟 오디언스 정보 없음 → CAMPAIGN_STRATEGIST에게 요청, TREND_ANALYST가 업종 일반 오디언스로 초안
├── 경쟁사 리스트 없음 → TREND_ANALYST가 웹 서치로 주요 경쟁사 3개 식별
├── 예산 범위 없음 → CPC 상한 없이 키워드 리서치 진행, 예산 가이드 별도 제안
└── 대상 채널 미지정 → 4개 채널 모두 리서치 (비용 추정은 채널별 제공)
```

### 6.2 시간 제약

```
긴급 리서치 (시간 부족) 시:
├── Phase 2: 각 에이전트가 핵심 결과물만 생성 (상세 분석 생략)
├── Phase 3: 크로스 리뷰를 리드가 대행 (3자 리뷰 → 1자 리뷰)
├── Phase 4: 생략 (피드백 미반영)
└── Phase 5: 통합 보고서 대신 3개 보고서 병렬 전달
```

### 6.3 팀원 부재

```
에이전트 1명 부재 시:
├── SEARCH_RESEARCHER 부재: TREND_ANALYST가 키워드 리서치 대행 (규모 축소)
├── AUDIENCE_RESEARCHER 부재: SEARCH_RESEARCHER가 기본 오디언스 설계 대행
└── TREND_ANALYST 부재: SEARCH_RESEARCHER + AUDIENCE_RESEARCHER가 경쟁/트렌드 분담
```

## 7. 커뮤니케이션 규칙

### 7.1 팀 내 커뮤니케이션

- 모든 산출물은 각 에이전트의 정의서(MD 파일)에 명시된 출력 JSON 스키마를 따른다
- 크로스 리뷰 피드백은 위의 피드백 양식을 사용한다
- 의견 충돌 시 리드(SEARCH_RESEARCHER)가 최종 판단한다
- 판단 기준: 데이터 근거가 있는 의견이 우선, 근거가 동등하면 리드 재량

### 7.2 팀 외 커뮤니케이션

- 외부 팀과의 소통은 리드(SEARCH_RESEARCHER)를 통해 이루어진다
- 단, 전문 영역 질문은 해당 에이전트가 직접 답변할 수 있다
  - 예: Creative Team이 오디언스 특성을 물으면 AUDIENCE_RESEARCHER가 직접 답변
- 핸드오프 문서는 반드시 리드가 최종 검수 후 전달한다
