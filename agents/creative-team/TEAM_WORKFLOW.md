# CREATIVE TEAM - 팀 워크플로우

> **팀**: Creative Team
> **구성원**: COPYWRITER, VISUAL_DIRECTOR, CREATIVE_REVIEWER
> **버전**: 1.0
> **최종 수정**: 2026-02-18

---

## 1. 팀 개요

### 1.1 팀 미션

키워드/오디언스 리서치 결과와 캠페인 전략을 기반으로,
4개 채널(네이버/카카오/메타/구글)에 최적화된 광고 카피와 크리에이티브 가이드를 제작하고,
법적/정책/품질 검수를 거쳐 Channel Team에 즉시 집행 가능한 크리에이티브 패키지를 산출한다.

### 1.2 팀 구성 및 역할

| 역할 | 에이전트 | 주요 책임 |
|------|---------|----------|
| 카피라이터 | COPYWRITER | 모든 채널의 텍스트 카피, USP 메시지화, A/B 카피 변형 |
| 비주얼 디렉터 | VISUAL_DIRECTOR | 이미지/영상 가이드, 채널 규격, thread-team 연동, 소재 트렌드 |
| 크리에이티브 심사관 | CREATIVE_REVIEWER | 법적/규제/정책/브랜드/품질 검수, A/B 가설 검증, 최종 승인 |

### 1.3 최종 패키징 역할

COPYWRITER가 최종 크리에이티브 패키지를 편집하여 Channel Team에 전달한다.
(카피 세트 + 비주얼 가이드가 하나의 통합 패키지로 합쳐짐)

## 3. Phase별 상세 가이드

### 3.1 Phase 1: 브리프 수령

**두 에이전트가 공통으로 확인**

```
수령 문서:
├── 키워드/오디언스 리서치 보고서 (handoff/keyword_team_to_creative_team.md)
│   ├── 주요 키워드 리스트 (카피 반영용)
│   ├── 검색 의도 정보 (카피 톤 결정)
│   ├── 오디언스 특성 (비주얼/카피 톤 결정)
│   └── 경쟁사 크리에이티브 인사이트
├── 캠페인 전략서 (CAMPAIGN_STRATEGIST로부터)
│   ├── 핵심 메시지/USP
│   ├── 캠페인 목표 (전환/인지/트래픽)
│   ├── 브랜드 가이드라인
│   └── 대상 채널
└── 제품/서비스 정보
    ├── 가격, 프로모션
    ├── 소셜 프루프 (리뷰, 수상)
    └── 제품 이미지/영상 소스

확인 사항:
├── 모든 채널이 명시되어 있는가?
├── USP가 명확한가? (불명확하면 CAMPAIGN_STRATEGIST에 질문)
├── 브랜드 가이드라인이 있는가? (없으면 자유도 높음)
├── 업종이 규제 업종인가? (건강/금융/주류 등 → CREATIVE_REVIEWER에 사전 알림)
└── 기존 성과 데이터가 있는가? (있으면 고성과 패턴 참고)
```

### 3.2 Phase 2: 병렬 제작

**COPYWRITER와 VISUAL_DIRECTOR가 동시에 독립적으로 제작**

| 에이전트 | 수행 작업 | 산출물 |
|---------|----------|--------|
| COPYWRITER | 채널별 카피 세트, A/B 변형, CTA 설계 | copy_package (JSON) |
| VISUAL_DIRECTOR | 크리에이티브 컨셉, 무드보드, 규격별 가이드, 스토리보드 | visual_creative_package (JSON) |

**병렬 제작 규칙**:
1. 두 에이전트는 같은 USP/핵심 메시지를 기반으로 독립 작업
2. COPYWRITER는 비주얼 요소를 기다리지 않고 카피 먼저 완성
3. VISUAL_DIRECTOR는 카피 완성을 기다리지 않고 컨셉/규격/무드보드 먼저 진행
4. 단, 텍스트 오버레이가 필요한 비주얼은 Phase 3에서 카피를 반영

### 3.3 Phase 3: 크로스 리뷰

**COPYWRITER와 VISUAL_DIRECTOR가 서로의 결과물을 리뷰**

#### 크로스 리뷰 포인트

| 리뷰어 | 대상 | 확인 사항 |
|--------|------|----------|
| COPYWRITER → VISUAL | 비주얼 가이드 | 비주얼 메시지가 카피 메시지와 일치하는가? 텍스트 오버레이 카피가 비주얼 레이아웃에 맞는가? |
| VISUAL → COPYWRITER | 카피 세트 | 카피 길이가 비주얼 레이아웃의 텍스트 영역에 적합한가? 영상 스크립트가 영상 구성과 동기화되는가? |

#### 크로스 리뷰 결과물

```
크로스 리뷰 후 동기화 사항:
├── 이미지 크리에이티브별 copy_id 매핑 확정
├── 텍스트 오버레이 카피 → 비주얼 레이아웃에 배치 확정
├── 영상 스토리보드 ↔ 영상 스크립트 씬별 매칭 확정
├── 메시지 정합성 확인 서명
└── 불일치 발견 시 수정 (이 단계에서 해결)
```

### 3.4 Phase 4: 심사관 검수

**CREATIVE_REVIEWER가 통합 검수 수행**

```
검수 입력:
├── COPYWRITER의 copy_package (크로스 리뷰 반영 후)
├── VISUAL_DIRECTOR의 visual_creative_package (크로스 리뷰 반영 후)
├── 브랜드 가이드라인
├── 업종 정보 (규제 업종 여부)
└── A/B 테스트 변형 (카피 + 비주얼 모두)

검수 프로세스:
1. 법적/규제 검수 (최우선)
2. 채널 정책 적합성 검수
3. 브랜드 일관성 검수
4. A/B 가설 논리성 검증
5. 카피 최종 점검 (오탈자, 자연스러움, 글자수)
6. 비주얼 규격 점검 (사이즈, 텍스트 비율, 세이프존)
7. 저작권/상표권 점검

검수 결과:
├── PASS → Phase 5로 진행
├── REVISE → 수정 지시서 발행
│   ├── 카피 이슈 → COPYWRITER에게 반환
│   ├── 비주얼 이슈 → VISUAL_DIRECTOR에게 반환
│   ├── 양쪽 이슈 → 양쪽 모두에게 반환
│   └── 수정 후 재검수 (최대 2회)
└── REJECT → TEAM_LEAD 에스컬레이션
```

#### 수정 사이클 규칙

```
수정 사이클 관리:
├── 1차 수정: REVISE 판정 → 수정 지시서 기반 수정 → 재검수
├── 2차 수정: 재검수에서도 REVISE → 추가 수정 → 최종 재검수
├── 2회 초과: TEAM_LEAD 에스컬레이션 (프로세스/브리프 문제 가능성)
└── 무한 루프 방지: 심사관은 수정 방향을 구체적으로 제시해야 함

수정 지시서 양식:
{
  "issue_id": "LEG-001",
  "severity": "critical|major|minor",
  "target_agent": "COPYWRITER|VISUAL_DIRECTOR",
  "item": "문제가 된 구체적 카피 또는 크리에이티브 ID",
  "issue": "무엇이 문제인가 상세 설명",
  "regulation_or_policy": "해당 법규/정책 조항",
  "fix_direction": "어떻게 수정해야 하는가",
  "alternative_example": "대안 카피/비주얼 예시 (가능 시)"
}
```

### 3.5 Phase 5: 최종 패키징

**COPYWRITER가 패키징 담당**

```
최종 패키지 구성:
├── 1. 카피 세트 (채널별/포맷별 정리)
│   ├── 네이버 카피 (파워링크/쇼핑검색/브랜드검색)
│   ├── 카카오 카피 (디스플레이/비즈보드/메시지)
│   ├── 메타 카피 (피드/리일스/스토리)
│   └── 구글 카피 (RSA/디스플레이/YouTube)
│
├── 2. 비주얼 크리에이티브 가이드
│   ├── 크리에이티브 컨셉 + 무드보드
│   ├── 채널별 이미지 가이드
│   ├── 영상 스토리보드
│   └── 규격 매핑표
│
├── 3. 카피-비주얼 매핑표
│   └── copy_id ↔ creative_id 대응표
│
├── 4. A/B 테스트 계획
│   ├── 카피 변형 + 가설
│   ├── 비주얼 변형 + 가설
│   └── 측정 지표/기간
│
├── 5. 검수 승인서
│   └── CREATIVE_REVIEWER의 PASS 확인
│
└── 6. 주의사항
    ├── 법적 주의 (해당 시)
    ├── 채널 심사 예상 이슈
    └── 시즌/타이밍 관련 노트

전달: handoff/creative_team_to_channel_team.md 양식으로 Channel Team 핸드오프
```

## 5. 품질 기준

### 5.1 팀 전체 품질 체크리스트

| # | 항목 | 기준 | 확인 주체 |
|---|------|------|---------|
| 1 | 모든 채널의 카피가 완성되었는가 | 네이버/카카오/메타/구글 4채널 전부 | COPYWRITER |
| 2 | 모든 채널의 비주얼 가이드가 완성되었는가 | 4채널 x 포맷별 규격 가이드 | VISUAL_DIRECTOR |
| 3 | 크로스 리뷰가 완료되었는가 | 카피↔비주얼 정합성 확인 | 양쪽 |
| 4 | 법적/규제 검수를 통과했는가 | CREATIVE_REVIEWER PASS 판정 | CREATIVE_REVIEWER |
| 5 | 채널 정책 적합성이 확인되었는가 | 채널별 심사 기준 충족 | CREATIVE_REVIEWER |
| 6 | 글자수가 모두 검증되었는가 | 채널별 정확한 글자수 이내 (구글 한글 포함) | CREATIVE_REVIEWER |
| 7 | A/B 테스트 가설이 논리적인가 | 가설+변수 통제+측정 지표 완비 | CREATIVE_REVIEWER |
| 8 | 오탈자가 0건인가 | 최종 점검 완료 | CREATIVE_REVIEWER |
| 9 | 카피-비주얼 매핑표가 완성되었는가 | copy_id ↔ creative_id 대응 | COPYWRITER |
| 10 | 핸드오프 문서가 준비되었는가 | Channel Team용 패키지 완성 | COPYWRITER |

### 5.2 품질 게이트

```
Phase 2 → Phase 3 게이트:
├── COPYWRITER: 최소 채널당 3세트 카피 + A/B 변형 1개
├── VISUAL_DIRECTOR: 최소 채널당 1개 크리에이티브 가이드 + 무드보드
└── 부족 시 Phase 2 연장

Phase 3 → Phase 4 게이트:
├── 카피-비주얼 메시지 정합성 확인 완료
├── 텍스트 오버레이 ↔ 레이아웃 동기화 완료
└── 불일치 시 Phase 3에서 해결

Phase 4 → Phase 5 게이트:
├── CREATIVE_REVIEWER PASS 판정
├── 모든 수정 사이클 완료
└── REVISE 항목 0건
```

## 6. 예외 처리

### 6.1 긴급 크리에이티브 (시간 부족)

```
긴급 모드 프로세스:
├── Phase 2: COPYWRITER 우선 (카피 먼저), VISUAL_DIRECTOR는 기본 규격 가이드만
├── Phase 3: 크로스 리뷰 축소 (메시지 정합성만 빠르게 확인)
├── Phase 4: 심사관 검수를 법적/정책 필수 항목만 (오탈자/가설 검증 축소)
└── Phase 5: 카피 위주 패키지 (비주얼은 후속 전달)

주의: 긴급 모드에서도 법적/규제 검수는 절대 생략하지 않는다.
```

### 6.2 심사관 REJECT 시

```
REJECT 대응 프로세스:
1. CREATIVE_REVIEWER가 상세 반려 사유서 작성
2. TEAM_LEAD에게 즉시 에스컬레이션
3. TEAM_LEAD가 브리프/전략 수준의 문제인지 판단
4. 필요 시 CAMPAIGN_STRATEGIST와 협의
5. 재작업 방향 확정 후 Phase 2부터 재시작
```

### 6.3 thread-team 소재 지연 시

```
thread-team 지연 대응:
├── 비주얼 가이드(텍스트 기반)는 먼저 Channel Team에 전달
├── 실제 소재는 thread-team 완료 후 별도 전달
├── Channel Team에 "소재 도착 예정일" 명시
└── 카피는 예정대로 출고 (텍스트 광고 먼저 집행 가능)
```

## 7. 커뮤니케이션 규칙

### 7.1 팀 내 커뮤니케이션

- 모든 산출물은 각 에이전트 정의서의 출력 JSON 스키마를 따른다
- CREATIVE_REVIEWER의 피드백은 반드시 수정 지시서 양식을 따른다
- 크로스 리뷰 시 구두 피드백이 아닌 문서화된 피드백을 남긴다
- 의견 충돌 시 CREATIVE_REVIEWER의 법적/정책 판단이 우선

### 7.2 팀 외 커뮤니케이션

- Channel Team과의 소통은 COPYWRITER (패키징 담당)를 통해 이루어진다
- thread-team과의 소통은 VISUAL_DIRECTOR를 통해 이루어진다
- 법적 이슈 관련 질문은 CREATIVE_REVIEWER가 직접 답변할 수 있다
- 핸드오프 문서는 COPYWRITER가 최종 편집 후 전달한다
