# Channel Team - 팀 워크플로우

## 팀 개요

채널 팀(Channel Team)은 네이버, 카카오, 메타, 구글 4개 광고 채널의
캠페인 세팅/실행/최적화/트러블슈팅을 담당하는 실행 조직이다.

각 채널 전문가가 독립적으로 자기 채널을 운영하면서도,
크로스 채널 시너지를 위해 정기적으로 동기화한다.

### 팀 구성

| 역할 | 에이전트 | 담당 채널 |
|------|---------|----------|
| 네이버 전문가 | NAVER_SPECIALIST | 네이버 검색광고 (파워링크, 쇼핑검색, 브랜드검색, 파워콘텐츠, GFA) |
| 카카오 전문가 | KAKAO_SPECIALIST | 카카오 모먼트 (디스플레이, 비즈보드, 친구톡/알림톡, 다음 검색) |
| 메타 전문가 | META_SPECIALIST | 메타 (인스타그램, 페이스북, 리일스, DPA, Advantage+) |
| 구글 전문가 | GOOGLE_SPECIALIST | 구글 (검색, GDN, YouTube, 쇼핑, P-Max) |

## 협업 모델: 독립 실행 + 크로스 채널 동기화

### 기본 원칙

```
1. 독립 실행 원칙
   - 각 채널 전문가는 자기 채널의 캠페인을 독립적으로 세팅/운영한다
   - 다른 채널 전문가의 승인 없이 자체 채널 작업을 진행할 수 있다
   - 단, 전략서(CAMPAIGN_STRATEGIST)와 입찰 전략(BID_OPTIMIZER)은 반드시 수령 후 작업

2. 크로스 채널 동기화 원칙
   - 채널 간 메시지 일관성을 유지한다
   - 크로스 채널 어트리뷰션 데이터를 공유한다
   - 채널 간 예산 재배분이 필요한 경우 팀 차원에서 BID_OPTIMIZER에 요청한다

3. 트러블슈팅 협업 원칙
   - 한 채널의 이슈가 다른 채널에 영향을 미칠 수 있으면 관련 전문가에게 즉시 공유한다
   - 예: 랜딩 페이지 장애 → 모든 채널에 영향 → 전체 팀 공유
```

## 워크플로우 1: 캠페인 생성 (신규 캠페인)

```
[캠페인 전략서 수령]
    │
    ├── CAMPAIGN_STRATEGIST로부터: 전략서, 채널 믹스, 예산 배분
    ├── KEYWORD_RESEARCHER로부터: 채널별 키워드/오디언스 데이터
    ├── AD_CREATIVE로부터: 채널별 광고 소재
    └── BID_OPTIMIZER로부터: 채널별 입찰/예산 전략
    │
    ▼
[Phase 1: 병렬 세팅] (각 채널 전문가 독립 실행)
    │
    ├── NAVER_SPECIALIST
    │   ├── 네이버 캠페인 구조 설계
    │   ├── 키워드 등록 (매치 타입별)
    │   ├── 광고 소재 등록
    │   ├── 확장 소재 설정
    │   ├── 전환 추적 코드 확인
    │   └── 네이버 론칭 체크리스트 실행
    │
    ├── KAKAO_SPECIALIST
    │   ├── 카카오 캠페인 구조 설계
    │   ├── 타겟팅/게재지면 설정
    │   ├── 소재 등록 (디스플레이/메시지)
    │   ├── 카카오 픽셀 확인
    │   ├── 메시지 템플릿 심사 제출
    │   └── 카카오 론칭 체크리스트 실행
    │
    ├── META_SPECIALIST
    │   ├── 메타 캠페인 구조 설계
    │   ├── 오디언스/게재위치 설정
    │   ├── 소재 등록 (이미지/영상/DPA)
    │   ├── 픽셀 + CAPI 확인
    │   ├── 도메인 인증/AEM 확인
    │   └── 메타 론칭 체크리스트 실행
    │
    └── GOOGLE_SPECIALIST
        ├── 구글 캠페인 구조 설계
        ├── 키워드/RSA/에셋 등록
        ├── 스마트 비딩 설정
        ├── GA4 연동/전환 확인
        ├── Merchant Center 확인 (쇼핑 시)
        └── 구글 론칭 체크리스트 실행
    │
    ▼
[Phase 2: 크로스 채널 검증] (동기화 포인트)
    │
    ├── 메시지 일관성 확인
    │   └── 각 채널의 핵심 메시지/USP가 동일한지 확인
    │
    ├── UTM 파라미터 정합성 확인
    │   └── 채널별 UTM 규칙 통일 (utm_source, utm_medium, utm_campaign)
    │
    ├── 랜딩 페이지 통합 확인
    │   └── 모든 채널에서 동일 랜딩 페이지 정상 작동 확인
    │
    └── 전환 추적 교차 검증
        └── 각 채널 픽셀/태그가 서로 간섭하지 않는지 확인
    │
    ▼
[Phase 3: 론칭]
    │
    ├── 각 채널 전문가: 광고 심사 제출
    ├── 심사 결과 확인 (승인/미승인)
    ├── 미승인 시 즉시 수정 후 재제출
    └── 전체 세팅 완료 → TEAM_LEAD에게 종합 보고
```

## 워크플로우 2: 일상 운영 (Daily Operations)

```
[일일 루틴]

AM 09:00 - 각 채널 전문가 독립 확인
    ├── NAVER_SPECIALIST: 네이버 성과 확인, 이상 여부 체크
    ├── KAKAO_SPECIALIST: 카카오 성과 확인, 차단율 체크
    ├── META_SPECIALIST: 메타 성과 확인, 빈도/소재 피로도 체크
    └── GOOGLE_SPECIALIST: 구글 성과 확인, 스마트 비딩 학습 상태 체크

→ 이상 발견 시: 즉시 DATA_MONITOR에 공유 + 자체 1차 대응

PM 15:00 - 일일 체크 (필요 시)
    ├── 예산 소진 속도 확인
    ├── 심사 미승인 건 대응 현황
    └── 크로스 채널 이슈 공유
```

## 워크플로우 3: 주간 크로스 채널 동기화

```
[주간 동기화 프로세스] (주 1회)

Step 1: 채널별 성과 요약 공유
    ├── 각 전문가가 자기 채널의 주간 성과 요약 작성
    │   ├── 핵심 지표: 노출, 클릭, 전환, CPA, ROAS
    │   ├── 전주 대비 변화
    │   └── 주요 이슈/발견사항
    └── DATA_MONITOR의 통합 대시보드 데이터 참고

Step 2: 크로스 채널 분석
    ├── 채널 간 성과 비교
    │   └── 동일 캠페인의 채널별 CPA/ROAS 비교
    ├── 어트리뷰션 데이터 공유
    │   └── 예: "메타에서 인지 → 네이버에서 전환" 경로 데이터
    ├── 메시지 일관성 확인
    │   └── 채널별 광고 메시지가 여전히 일관적인지
    └── 소재 성과 크로스 채널 비교
        └── 동일 메시지/USP의 채널별 성과 차이

Step 3: 액션 아이템 도출
    ├── 채널 간 예산 재배분 필요성 검토
    │   └── 필요 시 BID_OPTIMIZER에 요청
    ├── 저성과 채널 원인 분석 및 대응 방안
    ├── 고성과 전략의 크로스 채널 적용 검토
    │   └── 예: "네이버에서 잘 되는 키워드를 구글에서도 테스트"
    └── 다음 주 우선순위 합의

Step 4: TEAM_LEAD 보고
    └── 주간 채널팀 종합 보고서 제출
```

## 워크플로우 4: 트러블슈팅 프로토콜

### 이슈 분류 및 대응

```
[Level 1: 단일 채널 이슈] - 해당 전문가 독립 대응
    │
    ├── 광고 미승인 → 소재/키워드 수정 후 재심사
    ├── 입찰 경쟁 심화 → BID_OPTIMIZER와 협의
    ├── 소재 피로도 → AD_CREATIVE에 신규 소재 요청
    └── API 오류 → 기술적 디버깅 후 해결
    │
    대응 시한: 4시간 이내 1차 대응

[Level 2: 크로스 채널 영향 이슈] - 팀 전체 공유 후 협업 대응
    │
    ├── 랜딩 페이지 장애 → 전 채널 광고 일시 중단 검토
    ├── 전환 추적 장애 → 전 채널 트래킹 점검
    ├── 브랜드 이슈 (PR 위기 등) → 전 채널 광고 메시지 점검
    └── 대규모 예산 재배분 → 전 채널 예산 조정
    │
    대응 시한: 2시간 이내 팀 전체 공유, 4시간 이내 대응 방안 수립

[Level 3: 에스컬레이션] - TEAM_LEAD 보고
    │
    ├── 계정 정지/제한 (메타 계정 제한, 구글 계정 정지 등)
    ├── 법적 이슈 (상표 소송, 광고법 위반 지적 등)
    ├── 대규모 예산 초과 (일예산 200% 이상 소진)
    └── 성과 급격한 악화 (전일 대비 CPA 100%+ 상승)
    │
    대응 시한: 즉시 TEAM_LEAD 보고
```

### 트러블슈팅 보고 양식

```json
{
  "issue_report": {
    "reporter": "string (NAVER_SPECIALIST|KAKAO_SPECIALIST|META_SPECIALIST|GOOGLE_SPECIALIST)",
    "level": "number (1|2|3)",
    "channel": "string (naver|kakao|meta|google|all)",
    "issue_type": "string (disapproval|performance|technical|policy|budget)",
    "description": "string (이슈 상세 설명)",
    "impact": "string (영향 범위: 해당 캠페인만 / 채널 전체 / 전 채널)",
    "root_cause": "string (원인 분석, 파악된 경우)",
    "action_taken": "string (1차 대응 내용)",
    "cross_channel_impact": "boolean (다른 채널 영향 여부)",
    "support_needed": "string[] (필요한 지원: BID_OPTIMIZER, AD_CREATIVE, TEAM_LEAD 등)",
    "status": "string (investigating|in_progress|resolved|escalated)",
    "timestamp": "string (ISO 8601)"
  }
}
```

## 채널 간 데이터 공유 프로토콜

### UTM 파라미터 표준
```
채널별 UTM 규칙:
├── utm_source
│   ├── 네이버: naver
│   ├── 카카오: kakao
│   ├── 메타: meta (또는 facebook, instagram)
│   └── 구글: google
│
├── utm_medium
│   ├── 검색: cpc
│   ├── 디스플레이: display
│   ├── 소셜: social
│   ├── 동영상: video
│   ├── 쇼핑: shopping
│   └── 메시지: message
│
├── utm_campaign
│   └── {캠페인명}_{채널}_{날짜} 형식 통일
│
└── utm_content
    └── {소재ID} 또는 {소재유형}_{변형번호}
```

### 크로스 채널 어트리뷰션 데이터 공유
```
공유 데이터:
├── 전환 경로 데이터
│   ├── "메타(인지) → 네이버(검색) → 전환" 경로 비중
│   ├── "YouTube(인지) → 구글(검색) → 전환" 경로 비중
│   └── 각 채널의 퍼스트/라스트 터치 기여도
│
├── 오디언스 인사이트 공유
│   ├── 고성과 오디언스 프로필 (채널 간 공유)
│   ├── 리타겟팅 대상 공유 (개인정보보호법 범위 내)
│   └── 시간대/요일별 활성 패턴
│
└── 공유 주기
    ├── 일일: 이상 감지 시 즉시
    ├── 주간: 크로스 채널 동기화 시
    └── 월간: INSIGHT_ANALYST의 월간 리포트에 통합
```

## 성과 지표 관리

### 채널별 핵심 KPI

| 채널 | 주요 KPI | 보조 KPI |
|------|---------|---------|
| 네이버 | CPA, ROAS, CTR | 품질지수, 노출 점유율, PC/모바일 비율 |
| 카카오 | CPA, 도달수, 열람률 | 차단율, 채널 친구 순증, 클릭률 |
| 메타 | CPA, ROAS, CTR | 빈도, 도달률, 소재별 ThruPlay율 |
| 구글 | CPA, ROAS, CTR | 품질 점수, 최적화 점수, 검색 노출 점유율 |

### 채널 간 성과 비교 기준
```
비교 기준:
├── CPA 비교: 동일 전환 기준 (채널별 어트리뷰션 차이 고려)
├── ROAS 비교: 동일 매출 기준 (라스트 클릭 + 보조 전환 모두 고려)
├── 도달 효율: CPM 또는 도달당 비용
├── 인게이지먼트: 채널별 상호작용 비율
└── 주의: 채널별 역할이 다르므로 단순 CPA 비교는 위험
    ├── 메타/YouTube: 인지/고려 → 높은 CPA 허용 가능
    ├── 네이버/구글 검색: 전환 → 낮은 CPA 기대
    └── 카카오 메시지: CRM → 재구매율/LTV로 평가
```

## 외부 팀 연동

### 상위/하위 에이전트 연동

| 대상 | 연동 내용 | 주기 |
|------|----------|------|
| TEAM_LEAD | 세팅 완료 보고, 이슈 에스컬레이션, 주간 종합 보고 | 수시 + 주 1회 |
| CAMPAIGN_STRATEGIST | 전략서 수령, 채널별 실행 가능성/제약 피드백 | 캠페인 시작 시 |
| BID_OPTIMIZER | 채널별 입찰/예산 전략 수령, 실제 단가 피드백 | 캠페인 시작 시 + 주 1회 |
| AD_CREATIVE / CREATIVE_TEAM | 소재 수령, 규격 적합성 피드백, 신규 소재 요청 | 캠페인 시작 시 + 수시 |
| KEYWORD_RESEARCHER / RESEARCH_TEAM | 키워드/오디언스 데이터 수령, 성과 피드백 | 캠페인 시작 시 + 주 1회 |
| DATA_MONITOR | 채널별 성과 데이터 제공, 이상 감지 시 원인 파악 협업 | 일일 |
| INSIGHT_ANALYST | 채널별 심층 분석용 데이터/인사이트 제공 | 주 1회 |
